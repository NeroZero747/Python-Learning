A high-level summary of the topic

A Python module is a separate file where you store functions that any script in your project can share.

Think of your Python project as a workshop where every script shares the same toolbox. Instead of copying the same tool onto every workbench, you store it once and pick it up whenever you need it. A module is that toolbox, and Python is the language you use to fill it.

Key Concepts

Creating, importing, and selectively pulling functions from module files.

Creating a Module

A reusable Python file

.py file

A module is a plain Python file that holds functions your other scripts can borrow. Give the file a descriptive name and save it with a .py extension — that is all Python needs to recognise it as a module.

Module File Rules

Ends with .py utils.py sales_utils.py ✓
Lowercase + underscores SalesUtils.py ✗ → sales_utils.py
One theme per file calculations.py over misc.py
Python — sales_utils.py 
# sales_utils.py — a module file that holds sales functions
def calculate_total(price, quantity):  # a reusable function
    return price * quantity            # multiplies the two arguments

def apply_discount(total, rate):       # a second reusable function
    return total * (1 - rate)          # subtracts the discount percentage

Keep one theme per module file. A file called sales_utils.py should only hold sales-related functions — mixing unrelated logic makes the module harder to search and update later.

Importing a Module

Use the whole module at once

import

The import statement loads your module into the current script. You then access any function inside it by writing the module name, a dot, and the function name.

Import Syntax

import sales_utils Load the module
→
sales_utils.calculate_total() Call the function
→
Result returned
Python — main.py 
import sales_utils              # load the entire sales_utils module

total = sales_utils.calculate_total(10, 5)   # call a function from it
print(total)                   # prints: 50

discounted = sales_utils.apply_discount(50, 0.1)  # call a second function
print(discounted)              # prints: 45.0

The module file must be in the same folder as your main script. If Python cannot find sales_utils.py next to your script, it will raise a ModuleNotFoundError.

Specific Imports

Pick only what you need

from … import

The from … import syntax pulls one function directly into your script. You can then call it by name alone — no module prefix needed.

import vs from … import

import module from module import fn
Call style module.func() func()
Loads whole module ✓ Yes ✗ Just the function
Best when Using many functions Using one or two
Python — main.py 
from sales_utils import calculate_total    # import just this one function

total = calculate_total(10, 5)  # call it directly — no module prefix needed
print(total)                    # prints: 50

# You can also import multiple functions in one line
from sales_utils import calculate_total, apply_discount  # two at once

Use from … import when you only need one or two functions from a large module — it keeps your script shorter and makes it obvious which functions you are borrowing.

Code Examples

Hands-on code snippets to explore the concepts

01

Create a Module

Beginner Shop .py file

What This Does

This file saves two functions in a standalone Python file called shop_utils.py. Any script in the same folder can import this file and call either function without copying the code.

shop_utils.py 
# shop_utils.py — a reusable module for the shop project

def get_total(price, quantity):         # multiply price by number of items
    return price * quantity             # return the total cost

def apply_discount(total, percent):     # reduce a price by a percentage
    saving = total * (percent / 100)   # calculate the discount amount
    return total - saving              # return the discounted price

print(get_total(12, 4))                # quick test: 4 items at 12 each
Terminal $ python shop_utils.py
48

Save this file in your project folder — any script in that same folder can import it instantly.

02

Import a Module

Beginner Shop import dot notation

What This Does

This script loads every function from shop_utils.py using import, then calls each function using dot notation — shop_utils.function().

main_shop.py 
import shop_utils                              # load every function from shop_utils.py

shirt_cost = shop_utils.get_total(25, 3)      # call get_total using dot notation
print(shirt_cost)                             # 3 shirts at 25 each → 75

sale_price = shop_utils.apply_discount(shirt_cost, 10)  # apply a 10% discount
print(sale_price)                             # 75 with 10% off → 67.5
Terminal $ python main_shop.py
75
67.5

The shop_utils. prefix tells Python exactly which file a function comes from — helpful when you import several modules.

03

Pick Specific Functions

Beginner Shop from … import

What This Does

This script uses from … import to pull two functions directly into the current file. You call them by name alone — no module prefix needed.

daily_report.py 
from shop_utils import get_total          # import get_total directly — no prefix needed
from shop_utils import apply_discount     # import apply_discount the same way

jacket_cost = get_total(80, 2)           # call directly: 2 jackets at 80 each
print(jacket_cost)                        # 2 × 80 → 160

discounted = apply_discount(jacket_cost, 15)  # apply a 15% discount
print(discounted)                             # 160 with 15% off → 136.0
Terminal $ python daily_report.py
160
136.0

You can combine both imports on one line: from shop_utils import get_total, apply_discount.

SQL / Excel Comparison

How this topic compares across Python, SQL, and Excel

If you already use SQL stored procedures or Excel macros, you've been creating reusable logic for years. Python modules do exactly the same thing — they let you write a function once and call it from any script in your project.

Python
SQL
Excel
reusable logic container
Python module (.py file)

A plain Python file that holds functions any other script can import and use.

SQL stored procedure

A saved block of SQL that you can run by name from any query without rewriting.

Excel macro (VBA module)

A saved set of steps stored in a VBA module that any Excel workbook can run.

single reusable unit
Python function

A named block of code inside a module that takes inputs and returns a result.

SQL user-defined function

A custom SQL function you write once and call inside any SELECT.

Excel custom formula / UDF

A formula you build in VBA that works just like =SUM() in any cell.

loading the logic
Python import module

One line at the top of a script loads the whole module so you can call its functions.

SQL EXEC / CALL

You run a stored procedure by name with EXEC proc_name — no copying needed.

Excel =MacroName() / Run

You call a macro from any cell or button — Excel finds the code in the module automatically.

Same discount calculation, three tools

Python code

Python 
from shop_utils import apply_discount

sale_price = apply_discount(200, 10)
print(sale_price)

SQL query

SQL 
EXEC apply_discount
  @price    = 200,
  @percent  = 10;

Excel formula

Excel 
=ApplyDiscount(B2, 10)
' calls a VBA function
' stored in a module

All three apply a 10% discount to a price of 200 — the same result, just written in a different tool.

The idea of splitting logic into reusable, named units is the same across Python, SQL, and Excel. Python modules just give you a cleaner, more flexible way to organise that logic as your project grows.

Practice Exercises

Guided exercises to reinforce your learning

01

Build a Module

Beginner School def .py file

Your Task

You are building a helper file for a school grading system. Create a Python file called grade_utils.py that contains two functions: one that converts a raw score to a percentage, and one that returns a letter grade (Pass or Fail) based on that percentage. Run the file directly to test that both functions work correctly.

grade_utils.py 
# grade_utils.py — reusable grading functions

def to_percentage(score, total):         # convert a raw score to a percentage
    return round((score / total) * 100)  # divide, multiply by 100, round

def get_grade(percentage):               # decide Pass or Fail from a percentage
    if percentage >= 50:                 # 50 or above is a pass
        return "Pass"                    # return the pass label
    return "Fail"                        # anything below 50 is a fail

pct = to_percentage(38, 50)             # test: 38 out of 50
print(pct)                              # should print 76
print(get_grade(pct))                   # should print Pass
Terminal $ python grade_utils.py
76
Pass

Adding a quick test at the bottom of the module file is a common habit — it lets you run the file directly to check your functions work before importing them anywhere else.

02

Import and Use

Beginner School import dot notation

Your Task

Using the grade_utils.py module from exercise 1, create a separate file called report.py. Import the whole module using import grade_utils, then use dot notation to process scores for three students and print each result on its own line. Each output line should show the student's name, percentage, and grade.

report.py 
import grade_utils                              # load every function from grade_utils.py

alice_pct = grade_utils.to_percentage(45, 50)  # Alice scored 45 out of 50
alice_grade = grade_utils.get_grade(alice_pct) # decide Pass or Fail for Alice
print("Alice:", alice_pct, "%", alice_grade)  # print Alice's result

bob_pct = grade_utils.to_percentage(22, 50)    # Bob scored 22 out of 50
bob_grade = grade_utils.get_grade(bob_pct)     # decide Pass or Fail for Bob
print("Bob:", bob_pct, "%", bob_grade)        # print Bob's result

cara_pct = grade_utils.to_percentage(31, 50)   # Cara scored 31 out of 50
cara_grade = grade_utils.get_grade(cara_pct)   # decide Pass or Fail for Cara
print("Cara:", cara_pct, "%", cara_grade)     # print Cara's result
Terminal $ python report.py
Alice: 90 % Pass
Bob: 44 % Fail
Cara: 62 % Pass

Notice that report.py contains no calculation logic at all — it just calls functions from the module. This is the core benefit of modules: your main script stays short and readable.

03

Pick One Function

Beginner School from … import

Your Task

Create a file called quick_check.py that imports only the to_percentage function from grade_utils using from grade_utils import to_percentage. Use it to check the percentage for five exam scores (all out of 60) and print each result. You should be able to call to_percentage() directly without any module prefix.

quick_check.py 
from grade_utils import to_percentage   # import only the one function needed

scores = [54, 30, 42, 58, 21]          # five exam scores, all out of 60

for raw_score in scores:               # loop through each score

Terminal $ python quick_check.py
54 → 90 %
30 → 50 %
42 → 70 %
58 → 97 %
21 → 35 %

When you only need one function from a module, from … import keeps the code tidier — you skip the module name every time you call it.

Common Mistakes

Frequent errors and how to avoid them

Naming Your Module File the Same as a Python Library

Python loads your file instead of the real library, and calls to the library then fail.

Pitfall

When you save a file as math.py, Python finds it before the built-in math library — so import math loads your file instead. Any call to math.sqrt() then raises AttributeError. Rename your file to something specific to your project, such as shop_utils.py.

Wrong — shadows built-in math 

# math.py  -- conflicts with Python's built-in library
def add_items(a, b):
    return a + b

# main.py
import math               # loads math.py, not the built-in
result = math.sqrt(4)     # AttributeError: no attribute 'sqrt'

Correct — project-specific filename 

# shop_utils.py  -- unique name, no conflict
def add_items(a, b):
    return a + b           # custom function works fine

# main.py
import shop_utils          # loads shop_utils.py as intended
import math                # now loads the real built-in library
result = math.sqrt(4)      # 2.0 -- works correctly

Name modules after the task they perform — shop_utils.py, grade_utils.py, data_cleaning.py. Built-in names like math, os, and random are reserved by Python — reusing them as filenames silently swaps out the library.

Calling a Function Without Its Module Name

Python raises NameError because the bare function name is not in scope after a plain import.

Pitfall

Writing import shop_utils puts the module into scope — not its individual functions. Calling get_total(10, 3) without the prefix raises NameError: name 'get_total' is not defined. Either add the module prefix — shop_utils.get_total(10, 3) — or switch to from shop_utils import get_total to bring the function directly into scope.

Wrong — bare function name fails 

import shop_utils

total = get_total(10, 3)   # NameError: 'get_total' not defined

Correct — use the module prefix 

import shop_utils

# prefix tells Python where to look
total = shop_utils.get_total(10, 3)
print(total)                          # 30

Think of the module name like a workbook name in Excel. Writing get_total() is like a formula with no sheet reference — Python has no idea where to look. You need shop_utils.get_total(), or bring the function across with from shop_utils import get_total.

Placing the Module in a Different Folder from Your Script

Python raises ModuleNotFoundError because it only searches the folder where your script lives.

Pitfall

Python searches the same directory as the running script when resolving import statements. If shop_utils.py is in a subfolder while main.py is at the top level — or the reverse — Python raises ModuleNotFoundError: No module named 'shop_utils'. Keep both files in the same directory until you learn about Python packages.

Wrong — module in wrong directory 

# project/
#   main.py              -- script runs from here
#   utils/
#     shop_utils.py      -- module is one level down

import shop_utils  # ModuleNotFoundError: No module named 'shop_utils'

Correct — both files in same folder 

# project/
#   main.py              -- script
#   shop_utils.py        -- module at the same level

import shop_utils         # found -- Python looks in the same folder
total = shop_utils.get_total(10, 3)  # works correctly

While you are learning, always keep your module file in the same folder as your main script. Open your file manager and confirm that main.py and shop_utils.py sit side-by-side before you run the script.

Real-World Use

Practical applications in real-world workflows

Professional Python projects almost always split code across multiple files. A data team processing thousands of records each day cannot afford to duplicate calculation logic across every script. Modules give each script access to the same set of functions — write the function once, use it from any file in the project.

4 scripts,
one calculation

Your team has four scripts that all calculate a customer's total spend. You write get_total() once in shop_utils.py, and every script imports it rather than duplicating the formula.

returns order_total

A 10% discount,
applied everywhere

The pricing rule changes from 10% to 15%. You update apply_discount() in one file. Every script that imports it gets the new rate automatically.

returns sale_price

One project,
five focused files

Instead of a 500-line script, you split the project into data_cleaning.py, calculations.py, and export.py. Each file handles one job, and your main script simply imports what it needs.

returns clean_data

Without modules

Each of the four scripts contains its own copy of the calculation — any formula change must be updated in four separate files.

A discount rate change requires finding and editing every script individually, making inconsistencies almost certain.

A single 500-line script handles everything — no team member can read or change one part without risking another.

With modules

get_total() lives in one file — update it once and every script that imports it automatically gets the new logic.

apply_discount() holds the pricing rule — any change takes effect everywhere the function is called.

Splitting into focused modules means every team member edits only the file they own, with no risk of breaking another part of the project.
