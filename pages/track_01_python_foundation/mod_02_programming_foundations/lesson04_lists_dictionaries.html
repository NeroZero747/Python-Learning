Overview

A high-level summary of the topic

Py

A list stores a collection of values in order, and a dictionary gives each value a name so you can look it up directly.

Think of a filing cabinet with two sections: one section holds numbered folders — you find things by counting from the front — and the other uses labelled tabs so you can jump straight to "Name" or "City" without counting at all. A list is the numbered section, and a dictionary is the labelled section — Python is the language you use to build both.

List

The numbered section — folders filed in strict order

One value per slot, and the order never changes on its own.

Index

The slot number — reaches into exactly the right folder

Call out a position and the cabinet hands you that folder immediately.

Dictionary

The labelled section — skip counting, go straight by name

Ask for "City" and you get the right folder — no counting needed.

Key

The tab label — the word written on each divider

Every tab in the cabinet must have a unique word — no two can match.

If you have used VLOOKUP in Excel or a WHERE clause in SQL, you already think in lists and dictionaries — you just didn't have a Python name for them yet.

Key Takeaways

The most important ideas to remember

Indexes Start at Zero

Python counts list positions from 0, so the first item in any list is at index 0 and the last item sits at index length-minus-one.

Zero-based First is [0] Last is [n-1]

Choose List or Dictionary Wisely

Use a list when order matters, like a column of monthly figures; use a dictionary when each value needs a label, like a named column in a SQL table.

Ordered sequence Named lookup Data structure

Real Data Arrives in Both Forms

Every API response, database query result, and Excel import you process in Python arrives as a list, a dictionary, or a combination of both structures.

Key Concepts

Lists, indexes, dictionaries, and key-value pairs — with examples for each.

List

Ordered sequence of values

list

A list is an ordered, changeable sequence of values written inside square brackets [ ]. Python tracks the position of every item so you can read, add, or remove values at any time.

List Rules

Wrap items in [ ] [1, 2, 3] ✓
Separate items with , [1 2 3] ✗ → [1, 2, 3]
Any type is allowed [42, "text", True] ✓
Python 
sales = [120, 150, 200]      # three integers in a list
names = ["Ana", "Ben"]       # strings also work
first = sales[0]             # index 0 = first item → 120
last  = sales[-1]            # index -1 = last item → 200
print(len(sales))            # len() counts items → 3

Indexing starts at 0, not 1: the first item in any list is at sales[0], so a list of 3 items has valid indexes 0, 1, and 2 — accessing sales[3] causes an error.

Index

Position number for any item

int

An index is an integer position that identifies where an item sits inside a list. Python numbers positions from 0, so the first item is index 0, the second is index 1, and negative indexes count backwards from the end.

Index Quick Reference

scores[0] First item (leftmost)
scores[1] Second item
scores[-1] Last item (any length)
scores[-2] Second-to-last item
Python 
scores = [85, 90, 78, 95]   # list with four numbers
print(scores[0])             # first item → 85
print(scores[2])             # third item → 78
print(scores[-1])            # last item → 95

Accessing a position that does not exist raises an IndexError: válid indexes for a 4-item list run from 0 to 3 — use -1 to safely grab the last item regardless of length.

Dictionary

Key-value data lookup by name

dict

A dictionary is a collection of key-value pairs enclosed in curly braces { }. Each entry maps a unique string key to its value, letting you retrieve data by name rather than by position.

List vs Dictionary

list dict
Create [120, 150] {"jan": 120}
Access sales[0] sales["jan"]
Best for Ordered items Named fields
Python 
employee = {
    "name": "Maria",          # string key → string value
    "age": 34,                 # string key → integer value
    "active": True,           # string key → boolean value
}
print(employee["name"])       # look up by key → "Maria"

Dictionary keys must be unique: if you write the same key twice, Python silently keeps only the second value — check for typos when a value seems to disappear from your dictionary.

Key-Value Pair

The building block of every dictionary

key: value

A key-value pair is a single entry inside a dictionary — a string label on the left of a colon and its associated data on the right. Keys must be unique within one dictionary, but values can hold any Python type.

Anatomy of a Pair

"name" Key
:
"Alice" Value
→
Linked!
Python 
person = {"name": "Alice"}    # one key-value pair
person["city"] = "London"     # add a new key-value pair
person["name"] = "Bob"        # update an existing key's value
del person["city"]            # remove a key-value pair entirely

You can add a new key at any time: assign to it like a variable — person["email"] = "a@b.com" creates the key if it does not already exist in the dictionary.

Code Examples

Hands-on code snippets to explore the concepts

01

Shopping List

Beginner Everyday list index

What This Does

This script stores four items in a list, then uses index positions to read the first and last item, and len() to count them.

shopping_list.py 
shopping_list = ["milk", "eggs", "bread", "apples"]  # list of items to buy

first_item = shopping_list[0]   # index 0 = the first item
last_item  = shopping_list[-1]  # index -1 = always the last item

print("First item:",  first_item)          # → milk
print("Last item:",   last_item)           # → apples
print("Total items:", len(shopping_list))  # len() counts how many items → 4
Terminal $ python shopping_list.py
First item: milk
Last item: apples
Total items: 4

Index -1 always gives you the last item — even if the list grows from 4 items to 400.

02

Weekly Spending

Beginner Everyday sum() len()

What This Does

This script stores five daily spending amounts in a list, then uses sum() and len() to calculate a total and a daily average in two lines.

weekly_spending.py 
daily_spending = [4.50, 12.80, 6.00, 9.25, 15.40]  # amount spent each day this week

total_spent    = sum(daily_spending)           # sum() adds every number in the list
number_of_days = len(daily_spending)           # len() counts how many days there are
daily_average  = total_spent / number_of_days  # divide total by days for the average

print("Days tracked:",  number_of_days)  # → 5
print("Total spent:",   total_spent)     # → 47.95
print("Daily average:", daily_average)   # → 9.59
Terminal $ python weekly_spending.py
Days tracked: 5
Total spent: 47.95
Daily average: 9.59

Add another day's amount to the list and Python recalculates the total, count, and average automatically — no formula editing required.

Contact Card

Beginner Everyday dict key lookup

What This Does

This script stores five fields about a person in a dictionary, then reads each value by its key name — so you ask for "city" and get the city, no position number needed.

contact_card.py 
contact = {
    "name":  "Maria Santos",       # person's full name
    "phone": "07700 900123",       # mobile number
    "email": "maria@example.com",  # email address
    "city":  "Manchester",         # where they live
    "age":   31,                   # age in years (a number, not text)
}

print("Name: ",  contact["name"])   # look up the value stored under "name"
print("Phone:",  contact["phone"])  # look up the value stored under "phone"
print("City: ",  contact["city"])   # look up the value stored under "city"
print("Age:  ",  contact["age"])    # look up the value stored under "age"
Terminal $ python contact_card.py
Name: Maria Santos
Phone: 07700 900123
City: Manchester
Age: 31

A dictionary is the natural shape for any single record — each field label becomes a key, so you read data by name rather than counting column positions.

SQL / Excel Comparison

How this topic compares across Python, SQL, and Excel

If you already work with SQL tables or Excel spreadsheets, you already know what lists and dictionaries are — Python just gives them different names. A list is a column of values; a dictionary is a single row where each field has a name.

Python
SQL
Excel
ordered data
Python list

A Python list holds multiple values in a fixed order, just like a numbered column of cells.

SQL table column

A SQL column holds one type of value for every row in a table, in the order the rows were inserted.

Excel spreadsheet column

An Excel column (A, B, C…) stores a series of values you can reference by row number.

named record
Python dictionary

A Python dictionary groups related values together under named keys, like {"name": "Alice", "age": 30}.

SQL table row

A SQL row is one record in a table, where each column name acts as the field label for that record's values.

Excel spreadsheet row

An Excel row groups related cells across columns, where the header row acts as the field names for every data row below it.

read one item by position
Python list[index]

Write the list name followed by a number in square brackets, like members[0], to read the item at that position.

SQL LIMIT / OFFSET

SQL uses LIMIT 1 OFFSET n to skip to a specific row position and return just that one record.

Excel INDEX()

The Excel =INDEX(A:A, n) function returns the value at row n in a column, just as a list index does.

look up by name
Python dict["key"]

Write the dictionary name followed by a quoted key in square brackets, like provider["name"], to read that field's value.

SQL WHERE column = value

A SQL WHERE clause filters rows by matching a column value, letting you retrieve one specific record by name or ID.

Excel VLOOKUP()

Excel's =VLOOKUP() searches the first column of a table for a name and returns the value from another column in that row.

Same provider record, three tools

Python code

provider = {
    "name": "Dr. Lee",
    "specialty": "Cardiology",
    "npi": "1234567890"
}

All three retrieve a provider's name by matching on the NPI identifier — the same result, just written in a different tool.

Every time you work with a table in SQL or a sheet in Excel, you're already thinking in lists and dictionaries. Python just gives you the same structures as variables you can manipulate with code — once you know one, the other two feel immediately familiar.

Build a Roster

Beginner Members list len()

Your Task

A health plan has three enrolled members: "Alice Nguyen", "Carlos Rivera", and "Maria Santos". Create a list called enrolled_members that holds all three names in this order. Then print the first member using index [0], the last member using index [-1], and the total count using len() — your output should show three separate lines.

Using len() on a member list gives you the current enrollment count without counting manually — a real member portal might call this every time the roster is displayed to show how many active members a plan currently has.

02

Total Claims

Beginner Claims list sum()

Your Task

A provider submitted four claims this week with amounts of $120, $350, $85, and $240. Create a list called claim_amounts that holds all four values. Then use sum() to calculate the total billed and len() to count the claims, and print both results — your output should show two separate lines.

Common Mistakes

Frequent errors and how to avoid them

Using Parentheses Instead of Square Brackets Creates a Tuple, Not a List

Python uses ( ) for tuples and [ ] for lists — they behave differently and are not interchangeable.

Pitfall

Writing shopping_list = ("milk", "eggs", "bread") creates a tuple — a sealed sequence you cannot add to or change. Use square brackets [ ] to create a list that you can update anytime.

Wrong — parentheses → creates a tuple 

shopping_list = ("milk", "eggs", "bread")
# this is a tuple — items cannot be added or changed later

Correct — square brackets → creates a list 

shopping_list = ["milk", "eggs", "bread"]  # list: can be changed
shopping_list.append("butter")               # add a new item anytime

Think of square brackets as an open notepad you can always add to — parentheses seal the list permanently. In most everyday situations you want a list, so reach for [ ] by default.

Using Index 1 to Read the First Item Returns the Second Item Instead

Python counts list positions starting at 0, so the first item is always at [0], not [1].

Pitfall

Writing movies[1] to get the first movie silently returns the second movie — Python's zero-based indexing means the first item is always at position [0]. Use [0] for the first item and [-1] for the last.

Wrong — index 1 returns the second item 

movies = ["Interstellar", "The Matrix", "Inception"]
print(movies[1])
# prints "The Matrix" — not "Interstellar"

Correct — index 0 returns the first item 

movies = ["Interstellar", "The Matrix", "Inception"]
print(movies[0])   # index 0 → first movie: "Interstellar"
print(movies[-1])  # index -1 → last movie: "Inception"

A useful memory trick: the index equals the number of items before it. "Interstellar" has zero items before it, so it sits at [0]. This counting pattern appears throughout Python, so getting comfortable with it now saves a lot of confusion later.

Accessing a Dictionary Key That Doesn't Exist Raises a KeyError

Python stops immediately with KeyError if the key is not in the dictionary — it does not return None by default.

Pitfall

Writing contact["email"] when "email" is not a key raises KeyError: 'email' and stops the program. Use contact.get("email") to return None safely, or pass a default: contact.get("email", "Not provided").

Wrong — key not found → KeyError 

contact = {"name": "Maria Santos", "city": "Austin"}
print(contact["email"])
# KeyError: 'email' — program crashes

Correct — use .get() → returns safely 

contact = {"name": "Maria Santos", "city": "Austin"}
print(contact.get("email"))                    # returns None — no crash
print(contact.get("email", "Not provided"))    # returns "Not provided"

Not every contact has every field — someone might save a name and city but skip the email. Using .get() keeps your script running even when a field is missing, the same way a form lets some fields be optional.
