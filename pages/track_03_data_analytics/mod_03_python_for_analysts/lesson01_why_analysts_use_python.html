Lesson Objective

The goal and expected outcome of this lesson

What Python adds to analysts

Python automates repetitive tasks, handles millions of rows, and runs the same analysis every time without manual steps.

Key analytics libraries

Pandas, NumPy, and Matplotlib form the core analytics toolkit that replaces disconnected spreadsheet formulas with reusable code.

Python in a workflow

Python sits between raw data sources and final reports, transforming, cleaning, and summarizing data along the way.

When to choose Python

Python excels when tasks repeat regularly, datasets are large, or every step of an analysis must be auditable.

This lesson explains exactly where Python fits in an analyst's toolkit before you start using it in real work.

Overview

A high-level summary of the topic

Py

Think of Python like upgrading from a hand calculator to a programmable one — the same calculations run, but you write the formula once and it handles any size of input.

Your weekly task completed without any manual work.

Scale

If a task in Excel takes more than ten clicks and you repeat it weekly, a short Python script can replace it with a double-click.

Key Takeaways

The most important ideas to remember

Python Replaces Repetitive Clicks

A task that requires twenty manual filter and copy steps in Excel can be written as five lines of Python that run in under a second and produce identical output every time.

20 clicks → 5 lines Instant Consistent

Scripts Are Testable — Clicks Are Not

A Python script can be run on a small sample dataset and its output verified before it touches production data; an Excel macro cannot be unit-tested in the same way.

Testable Sample data Verification

Python Connects Where Excel Cannot

Python connects directly to live databases, REST APIs, and cloud storage buckets, pulling data into the analysis without any manual downloading, opening, or copy-pasting of files.

Databases REST APIs Cloud storage

Key Concepts

Automation, reproducibility, scalability, and the library ecosystem.

Automation

Let code do the repetitive work

concept

Automation means writing a script that performs a task for you. You run it once, and it processes hundreds of files or rows without manual clicks.

Manual vs Automated

Manual Automated
Speed ✗ Minutes per file ✓ Seconds per file
Errors ✗ Risk of typos ✓ Consistent every run
Effort ✗ Redo each time ✓ Write once, rerun forever
Python 
files = ["jan.csv", "feb.csv", "mar.csv"]  # list of reports
for f in files:                             # loop through each
    df = pd.read_csv(f)                     # load the file
    print(f, df.shape)                      # show row count

Start with the task you repeat most often. If you copy-paste the same Excel steps every week, that task is your first automation candidate.

Reproducibility

Rerun the same steps every time

concept

Reproducibility means anyone can rerun your script and get the same result. A Python script records every step. Manual Excel work cannot be replayed or audited.

Excel vs Python Script

Excel workflow Python script
Steps visible ✗ Hidden in clicks ✓ Written in code
Rerun ✗ Start over manually ✓ Run the script again
Audit trail ✗ None ✓ Full script history
Python 
df = pd.read_csv("sales.csv")              # step 1: load data
df = df[df["amount"] > 0]                   # step 2: filter rows
df.to_csv("cleaned.csv", index=False)        # step 3: save result

Save your script with a descriptive name. A file called clean_sales_data.py tells your future self exactly what it does.

Scalability

Handle millions of rows without crashing

concept

Scalability means your code handles large datasets the same way it handles small ones. Excel slows down above one million rows. Python processes tens of millions without crashing.

Excel vs Python

Excel Python
Max rows ✗ ~1,048,576 ✓ Limited by RAM only
Speed ✗ Freezes on large data ✓ Optimized for speed
Multiple files ✗ Open each manually ✓ Loop and combine
Python 
df = pd.read_csv("big_data.csv")            # 5 million rows
print(df.shape)                              # (5000000, 12)
total = df.memory_usage().sum() / 1e6        # memory in MB
print(f"Total: {total:.1f} MB")              # readable size

If your spreadsheet takes more than 30 seconds to open, it is a sign that you need Python instead.

Library Ecosystem

Pre-built tools for every analyst task

concept

A library is a package of pre-built code you install and import. Python has thousands of libraries. Analysts use a small core set for data, charts, and automation.

Common Analyst Libraries

Op Meaning Example
pandas Data loading and manipulation import pandas as pd
matplotlib Charts and visualizations import matplotlib.pyplot as plt
openpyxl Read and write Excel files pip install openpyxl
glob Find files by name pattern import glob
Python 
df = pd.read_csv("data.csv")                # pandas: load data
df["total"] = df["qty"] * df["price"]       # pandas: calculate
df.to_excel("report.xlsx", index=False)      # openpyxl: save Excel

Start with pandas. It covers 80% of analyst tasks. The other libraries become useful once you have your data loaded and cleaned.

Code Examples

Hands-on code snippets to explore the concepts

01

Automate a Calculation

Beginner Budgets pandas sum()

What This Does

This script loads a budget spreadsheet and calculates the total spend in one line. In Excel, you would need a SUM formula at the bottom of a column.

budget_total.py 
import pandas as pd  # load pandas

df = pd.read_csv("budget.csv")  # load budget data

total = df["amount"].sum()  # add up the amount column
print("Total spend:", total)   # display the result
Terminal $ python budget_total.py
Total spend: 48750.0

One line of pandas replaces the manual SUM formula you would normally type at the bottom of a spreadsheet column.

02

Summarize by Category

Beginner Sales groupby() pivot-table style

What This Does

This script groups rows by region and calculates the average — like a pivot table in Excel, but reusable and repeatable.

sales_summary.py 
import pandas as pd  # load pandas

df = pd.read_csv("sales.csv")  # load sales data

avg_by_region = df.groupby("region")["revenue"].mean()  # average per region
avg_by_region = avg_by_region.round(2)  # round to 2 decimal places
print(avg_by_region)
Terminal $ python sales_summary.py
region
East 1125.00
North 800.50
West 1275.25
Name: revenue, dtype: float64

Unlike a pivot table, this script gives you the same result every time you run it — no manual clicking required.

03

Use a Library

Beginner Reports import library ecosystem

What This Does

This script uses pandas and datetime together. Libraries give you ready-made tools so you do not need to write everything from scratch.

report_date.py 
import pandas as pd              # data analysis
from datetime import date         # today's date

df = pd.read_csv("report.csv")   # load report data
row_count = len(df)               # count rows
today = date.today()              # get today's date

print(f"Report: {row_count} rows as of {today}")
Terminal $ python report_date.py
Report: 340 rows as of 2026-03-31

Python has over 500,000 libraries — chances are someone has already built a tool for the task you are trying to automate.

01

Sum a Column

Beginner Donations sum() pandas

Your Task

A charity tracks donations in donations.csv with an amount column. Load the file and print the total amount donated.

total_donations.py 
import pandas as pd  # load pandas

df = pd.read_csv("donations.csv")  # load donation data

One line of pandas replaces a manual SUM formula in a spreadsheet — and you can rerun it any time new data arrives.

02

Count by Category

Beginner Tickets groupby() count

Your Task

Load support_tickets.csv. Group by priority and count how many tickets belong to each level. Print the result.

ticket_counts.py 
import pandas as pd  # load pandas

df = pd.read_csv("support_tickets.csv")  # load tickets

counts = df.groupby("priority")["ticket_id"].count()  # count per group
print(counts)
Terminal $ python ticket_counts.py
priority
High 42
Low 118
Medium 86
Name: ticket_id, dtype: int64

groupby() plus count() is the pandas equivalent of a COUNTIF pivot table in Excel.

03

Add Today's Date

Beginner Inventory datetime import

Your Task

Load stock.csv and print a message showing the row count and today's date. Use the datetime library to get the date.

stock_report.py 
import pandas as pd
from datetime import date

Common Mistakes

Pitfalls beginners hit when working with choosing Python for analysis work

Expecting Python to Replace Every Existing Tool

Python complements Excel and SQL — it does not eliminate the need for them.

Pitfall

Python is best for automation, large datasets, and reproducible workflows. Quick ad-hoc lookups are often faster in Excel or SQL. Use each tool where it excels.

Wrong — force Python everywhere 

# rewriting a simple 5-row lookup in Python
df = pd.read_csv("data.csv")          # overkill
print(df[df["ID"] == 42]["Name"])      # Excel was faster

Correct — right tool for the job 

# Python shines with repeatable large-scale work
df = pd.read_csv("transactions.csv")   # 500K rows
summary = df.groupby("Region").sum()    # instant aggregation
summary.to_excel("report.xlsx")         # automated export

Ask yourself: "Will I do this more than once?" or "Is the data too big for a spreadsheet?" If yes, use Python. If it is five rows and one-off, open Excel.

Writing Complex Code for a Task Excel Handles in Seconds

A 20-line script for a one-off sum wastes time Python is meant to save.

Pitfall

Python saves time through automation and repeatability. If a task takes 30 seconds in Excel and you will never repeat it, the Excel route is better.

Wrong — one-off in Python 

df = pd.read_csv("five_items.csv")
total = df["Price"].sum()
print(total)   # 3 lines for one number

Correct — automate the repeated task 

# automate a weekly report across 12 files
for month_file in monthly_files:              # loop each file
    df = pd.read_csv(month_file)              # load month data
    df.groupby("Team").sum().to_excel(         # aggregate + save
        f"summary_{month_file}")               # one file per month

Python's strength is not doing one thing faster — it is doing the same thing hundreds of times without you lifting a finger.

Jumping Straight Into Code Without Exploring the Data First

Errors appear deep in the pipeline because assumptions about the data were wrong.

Pitfall

Always start with df.head(), df.info(), and df.describe() before writing transformation code. These three commands reveal missing values, wrong types, and unexpected ranges.

Wrong — code blindly 

df = pd.read_csv("data.csv")
df["Revenue"] = df["Qty"] * df["Price"]  # crashes — Qty is text

Correct — explore first 

df = pd.read_csv("data.csv")   # load data
print(df.info())                # check types and nulls
print(df.describe())            # check numeric ranges
print(df.head())                # eyeball the first rows

Spending sixty seconds on info() and describe() saves sixty minutes of debugging later.

Real-World Use

How Python for analysts is used across real-world workflows

Imagine you're a data analyst at a mid-size company. Every week you spend hours copying SQL results into Excel, refreshing pivot tables, and emailing reports. Python doesn't replace SQL or Excel — it connects them and automates the steps in between, so you spend time on insight, not on plumbing.

Monthly report that
takes 4 hours manually

Every month you spend 4 hours querying, formatting, and emailing one report. The same pipeline written with pandas and openpyxl runs in under 2 minutes, automatically.

returns report.xlsx

Data from 3 different
database systems

Your analysis spans an Oracle, SQL Server, and Postgres database. One Python script connects to all three with pd.read_sql(), queries each, and combines the results — no switching tools.

returns combined_df

1.2 million rows
that crash Excel

Your transaction file has 1.2 million rows — Excel refuses to open it. pd.read_csv() loads the full file in seconds and your analysis runs without hitting a row limit.

returns full_df

Without Python

The 4-hour monthly report runs every time someone manually clicks through the same SQL queries, paste steps, and Excel formatting — and breaks if one step is interrupted.

Querying three different databases in a single analysis means logging into three separate clients, exporting three files, and manually merging them in Excel.

A file with 1.2 million rows simply cannot be opened in Excel — the analysis cannot happen until someone splits the file by hand into smaller chunks.

With Python

pandas and openpyxl script the entire 4-hour workflow — one command runs in under 2 minutes with no human steps.

pd.read_sql() connects to any database — one script queries Oracle, SQL Server, and Postgres in sequence and combines all three results.

pd.read_csv() loads 1.2 million rows in seconds — no row limit, no file splitting, no workarounds.
