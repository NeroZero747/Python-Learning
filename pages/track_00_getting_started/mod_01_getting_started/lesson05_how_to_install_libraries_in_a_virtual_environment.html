Overview

A high-level summary of the topic

pip

Python’s built-in language is intentionally small — the real power comes from libraries that you install for each project.

Think of a library like a power tool — you don’t build a drill from scratch every time you need to hang a shelf, you just pick one up and use it. Libraries work the same way: someone has already written the code to analyse data, call an API, or draw a chart, and you simply install it and use it in your own project.

Libraries live on PyPI (the Python Package Index) and are installed with a single terminal command using pip, Python’s built-in package manager.

Commonly Used Libraries

Pandas

Data analysis and manipulation

NumPy

Numerical operations and arrays

Requests

Sending HTTP requests to APIs

Matplotlib

Charts and data visualisation

Installing a Library

Run pip install followed by the library name. pip fetches it from PyPI and places it in your active environment.

terminal 
pip install pandas

Always activate your virtual environment before running pip install. This keeps libraries scoped to the current project and prevents conflicts with other projects on your machine.

Key Concepts

Core terms and definitions for this topic

Library

Definition

A library is a bundle of pre-written Python code you import into your project to add specific capabilities. Rather than coding everything from scratch, you install a library and call its ready-made functions directly.

Solves a specific problem Community maintained Ready to use instantly
Python 
import pandas as pd          # import the library
df = pd.read_csv("data.csv") # use its functions immediately

pip

Definition

pip is Python’s built-in package manager. It connects to PyPI (the Python Package Index) and downloads any published library with a single terminal command.

Common Commands

pip install <name> Download and install a library
pip uninstall <name> Remove an installed library
pip list Show all installed libraries
pip freeze > requirements.txt Save dependencies to a file

Dependency

Definition

A dependency is any library your project requires in order to run. Tracking dependencies lets you (or anyone else) recreate the exact environment your code needs.

Tracking with requirements.txt

Run pip freeze > requirements.txt to save all current dependencies. Share that file with your project so others can install the same versions in one command.

requirements.txt 
pandas==2.1.0
numpy==1.25.2
requests==2.31.0
Install all dependencies at once 
pip install -r requirements.txt

Follow Along

Install a library inside your virtual environment

1

Activate Your Virtual Environment

Before installing anything, make sure your virtual environment is active. You will see (venv) at the start of your terminal prompt when it is on.

terminal 
# Mac / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
2

Install a Library with pip

Use pip install followed by the library name. pip downloads it from PyPI and places it inside your virtual environment.

terminal 
pip install pandas

You can install multiple libraries at once: pip install pandas numpy requests

3

Confirm the Library Installed

Run pip list to see every library installed in the active environment and confirm yours is there.

terminal 
pip list
Output
Package Version --------- ------- pandas 2.1.0 numpy 1.25.2 requests 2.31.0
4

Import and Use the Library in Python

Once a library is installed, import it at the top of your Python file. It is then available for the entire script.

analysis.py 
import pandas as pd

data = {"Name": ["Alice", "Bob", "Carol"], "Score": [92, 85, 78]}
df = pd.DataFrame(data)

print(df)
Output
Name Score 0 Alice 92 1 Bob 85 2 Carol 78

The alias as pd lets you type pd instead of pandas throughout your code — a common convention.

Troubleshooting: pip Connection Error

Fix pip install failures caused by corporate network settings on Windows

Windows only. This fix applies to machines on a corporate network where pip cannot reach PyPI directly. If you are on a Mac or a personal machine, you are unlikely to encounter this issue.

What’s happening?

When you run pip install on a corporate machine, pip may try to reach PyPI directly but get blocked by the network proxy. You will see a connection error or time-out message in the terminal.

The fix is to create a pip.ini configuration file that points pip to the corporate package mirror instead.

Fix: Create a pip.ini Configuration File

A one-time setup — pip will use the corporate mirror from this point forward

1

Navigate to the pip config folder

Open File Explorer and go to the path below — replace [LoginID] with your Windows username. If the pip folder does not exist, create it.

C:\Users\[LoginID]\AppData\Roaming\pip
2

Create a file called pip.ini

Inside the pip folder, create a new file named exactly pip.ini. In File Explorer right-click → New → Text Document, then rename it — make sure to remove the .txt extension.

3

Open the file in VS Code or Notepad and paste this configuration

Replace the two placeholder URLs with the internal package mirror URLs provided by your IT or platform team.

pip.ini 
[global]
timeout = 120
index-url = [company image url]
trusted-host = pypi.org files.pythonhosted.org
extra-index-url = [company second image url]

Swap [company image url] and [company second image url] with the actual URLs. If you’re unsure what they are, ask your IT or data platform team.

4

Save the file, restart VS Code, and try again

Save pip.ini, fully close and reopen VS Code, then run your pip install command again. pip will now route through the corporate mirror instead of trying to reach PyPI directly.
