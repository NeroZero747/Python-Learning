Overview

A high-level summary of the topic

Py

Python gives you two ways to run your code — a script that runs from top to bottom in one shot, and a notebook that lets you run and inspect code one section at a time.

Think of a script like a dishwasher — you load it, press start, and it runs the whole cycle without stopping. A notebook is more like washing dishes by hand: you do one item at a time, check the result, and decide what to do next. Both get the job done, but the right choice depends on what you’re trying to accomplish.

Python Script

.py
  • Runs the entire file from top to bottom
  • Great for automation and scheduled jobs
  • Run from the terminal or VS Code play button
  • Standard format for production code

Jupyter Notebook

.ipynb
  • Runs code one cell at a time
  • Great for data exploration and visualization
  • Shows output directly below each cell
  • Mix code, notes, and charts in one file

By the end of this lesson you’ll know when to reach for each one — and how to run both in VS Code and the terminal.

Key Takeaways

The most important ideas to remember

Scripts Run Your Entire Program in One Go

A .py script runs from the first line to the last every time you execute it. There are no pauses, no checkpoints — just one complete run. This makes scripts ideal for repeatable, predictable tasks.

Automation Data pipelines Scheduled jobs Production code

Notebooks Let You Run Code One Cell at a Time

A .ipynb notebook is split into cells. You run each cell independently, see its output immediately, and then decide what to do next. This makes notebooks perfect for exploring data and testing ideas.

Data exploration Interactive output Visualizations Experimentation

Use the Right Tool for the Right Job

Neither tool is inherently better — they serve different stages of your workflow. Use a notebook when you’re figuring things out, and switch to a script when you’re ready to run, repeat, or share your work reliably.

Key Concepts

Core terms and definitions for this topic

Python Script

Definition

A Python script is a plain text file saved with a .py extension. When you run it, Python reads every line from top to bottom and executes each one in order.

Runs in sequence Saved as .py Schedulable
greet.py 
name = "Alice"
print("Hello,", name)
print("Script complete")

Jupyter Notebook

Definition

A Jupyter Notebook is an interactive document made up of cells. Each cell can hold Python code, and you run them one at a time — seeing the output immediately below before moving on. Notebooks are saved as .ipynb files.

Runs cell by cell Saved as .ipynb Inline output
analysis.ipynb
In [1]: 
import pandas as pd
In [2]: 
df = pd.read_csv("sales.csv")
print(df.head())
Out [2]:

3 2024-01-04 Widget C 20 600.00

Python Interpreter

Definition

The Python interpreter is the program that actually runs your code. When you type python filename.py in the terminal, the interpreter reads every line and executes it. If something goes wrong, it stops and shows you an error message.

Reads .py files Executes line by line Shows errors
Terminal 
python greet.py
Output
Hello, Alice
Script complete

Decision Flow

A visual walkthrough of the logic and process

Whether you run a script or a notebook, Python always follows the same five steps — every single time. Understanding this flow makes it much easier to read errors and debug problems.

Step 1

Write your code

Open your editor and type your Python instructions. Each line tells Python to do something specific — print text, do a calculation, read a file, and so on. Nothing actually runs yet; this is just you composing the instructions.

VS Code Jupyter Notebook Any text editor
Step 2

Save the file

Before Python can run anything, the file needs to be saved to disk. The file extension tells Python — and your tools — what kind of file it is. Most editors auto-save, but it's always worth checking.

.py

Python script

.ipynb

Jupyter Notebook

Step 3

Trigger execution

This is where you actually tell Python to go. How you do it depends on your tool — but all three methods do the same thing: hand your file to the Python interpreter.

Terminal

python hello.py

VS Code

Click the ▶ Run button (top-right) or press Ctrl+F5

Jupyter Notebook

Press Shift+Enter inside a cell

Step 4

Python reads and executes — line by line

The Python interpreter works through your code from the very first line to the last, executing each instruction in order. It cannot skip ahead. If line 3 depends on something from line 1, that's fine — but line 1 must come first.

This top-to-bottom order is why the sequence of your code matters — defining a variable after you use it will cause an error.

Step 5

See the results (or the error)

Once execution finishes, the output appears — either in the terminal or directly below your cell in Jupyter. If Python hit a problem, it stops immediately and prints an error message that includes the exact line number where things went wrong.

Success

Output is printed to the terminal or appears below the cell

Error

Python stops and shows the line number + a message explaining what went wrong

How-To Guide

Step-by-step: terminal, VS Code button, and Jupyter Notebook

1
Method 1

Run a script from the terminal

Open a terminal, navigate to the folder that contains your file, then type python followed by the filename and press Enter. The output appears immediately.

terminal 
cd my-project
python hello.py

Getting "No such file or directory"? You're in the wrong folder. Run ls (Mac/Linux) or dir (Windows) to see where you are.

2
Method 2

Run a script using VS Code

Open your .py file in VS Code, then click the ▶ Run button in the top-right corner — or use the keyboard shortcut. VS Code opens an integrated terminal at the bottom and runs the file automatically.

Windows / Linux

Ctrl + F5

Mac

Cmd + F5

The Python extension (by Microsoft) must be installed in VS Code for the Run button to appear. Install it from the Extensions panel (Ctrl+Shift+X).

3
Method 3

Run code in a Jupyter Notebook cell

A notebook is split into cells — small blocks of code you run one at a time. The output appears directly below each cell. To open one in VS Code, press Ctrl+Shift+P and search "Jupyter: Create New Jupyter Notebook". Click a cell, type your code, then press Shift+Enter to run it.

Keyboard shortcuts

Shift + Enter

Run cell and move to the next one

Ctrl + Enter

Run cell and stay on it

Alt + Enter

Run cell and insert a new cell below

In [*]: means the cell is still running. In [1]: means it finished — the number shows the order cells were run.

Code Examples

Short reference snippets for each method — script, VS Code, and notebook

01

Example 1 — Run a Python Script from the Terminal

Beginner

What This Does

Create a file called hello.py, add a print() statement, and run it from the terminal. This is the most direct way to execute a Python script.

example_1_running_a_python_script.py 
print("Hello Python")
Terminal $ python example_1_running_a_python_script.py
Hello Python

The terminal shows Hello Python immediately after running. If you see an error, double-check that you saved the file and that the filename matches exactly.

02

Example 2 — Run a Script Using the VS Code Play Button

Beginner

What This Does

Open a .py file in VS Code, then click the ▶ Run button in the top-right corner. VS Code opens an integrated terminal and runs the script for you automatically.

example_2_running_code_in_vs_code.py 
print("Running Python inside VS Code")

VS Code opens the integrated terminal at the bottom and shows the output there. You can also press Ctrl+F5 (Windows) or Cmd+F5 (Mac) to run without debugging.

03

Example 3 — Run Code in a Jupyter Notebook Cell

Beginner

What This Does

In a Jupyter Notebook each block of code lives in its own cell. Press Shift+Enter to run a cell — output appears directly below it, leaving all other cells untouched.

analysis.ipynb
In [1]: 
name = "Python"

print("Learning", name)
Out [1]:
Learning Python

Unlike a script, only the cell you pressed Shift+Enter on is executed. The other cells stay untouched until you run them. This is what makes notebooks great for step-by-step exploration.

Real-World Use

Practical applications in real-world workflows

In real data teams, scripts and notebooks aren't competing tools — they're used at different stages of the same workflow. Knowing when to use each one is one of the first practical skills you'll build.

Use a Jupyter Notebook when…

  • You're exploring a new dataset for the first time
  • You want to see charts and output inline as you go
  • You're testing ideas and trying different approaches
  • You're writing up an analysis to share with others

Use a Python Script when…

  • You need to run the same code automatically on a schedule
  • You're building an ETL pipeline or data ingestion job
  • The code needs to run reliably in a server or CI/CD pipeline
  • You're sharing code with people who just need to run it
