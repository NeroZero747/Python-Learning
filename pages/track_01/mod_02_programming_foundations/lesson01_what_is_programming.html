Overview

A high-level summary of the topic

Py

Programming is the process of writing instructions that tell a computer what to do.

Think of a computer like a very precise chef's assistant: it will follow your recipe step by step, in exact order, without skipping or improvising — but only if you write out every instruction clearly. A program is that recipe, and Python is the language you use to write it.

The program

The recipe — your complete list of steps

A program is a written sequence of instructions you prepare in advance. The computer reads from the first line and works through every step in order.

The computer

The assistant — follows steps without improvising

Your computer executes each instruction exactly as written — never skipping, reordering, or making assumptions. Clear and precise instructions are essential.

Automation

Batch prep — repeat the same steps on any scale

Once your recipe is written, Python can run it on 10 rows of data or 10 million rows with no extra effort from you.

Python

The language — how you write the recipe

Python is one specific programming language. It's popular among data professionals because its syntax reads almost like plain English.

You already use tools that run step-by-step instructions — every Excel formula and SQL query is a form of programming. Python gives you a more powerful and flexible way to do the same thing.

Key Takeaways

The most important ideas to remember

Programs Are Sets of Instructions

Unlike an Excel formula that recalculates a single cell, a program is a self-contained workflow — it can load data, clean it, calculate results, and export a report, all in one run without you touching the keyboard.

Step-by-step Executed in order No improvising

Programming Automates Work

Every time you manually copy-paste data, reformat a spreadsheet, or re-run the same SQL query, that's a candidate for automation. A Python script replaces that manual effort permanently — you run it once and the work is done.

Write once, run forever Repeatable Saves hours

Programs Can Make Decisions

SQL filters rows and Excel applies formulas, but neither can easily react to what it finds. Python can — it reads a value, evaluates a condition, and chooses a different action depending on the result, all without you intervening.

Conditional logic if / else Dynamic behaviour

Key Concepts

Core terms and definitions for this topic

Instruction

Definition

An instruction is a single command you give to a computer — one specific action it should perform right now. Every program is made up of instructions executed one after another.

You already do this in Excel: =SUM(A1:A10) is an instruction that tells Excel to add up a column. In Python, you write instructions using plain words instead of formula syntax.

Python instruction — one line, one action:

Python 
print("Hello, world!")   # This is one instruction — display text on screen

One instruction = one action. Python runs it immediately when the line is reached — no button clicks, no pressing Enter in a cell.

Program

Definition

A program is a complete set of instructions that work together to accomplish a task. While a single instruction does one thing, a program chains many instructions into a workflow — carried out in order, from the first line to the last.

In SQL, each query is a separate action. A Python program replaces a whole sequence of queries — load the data, clean it, calculate totals, and export the result — all in one automated run.

A simple 3-step program:

Python 
print("Step 1: Loading data...")       # instruction 1
print("Step 2: Calculating totals...")  # instruction 2
print("Step 3: Export complete.")       # instruction 3
Line 1 runs first Then line 2 Then line 3

Programming Language

Definition

A programming language is a set of rules and vocabulary that lets you write instructions a computer can understand. Think of it like a spoken language — French, Spanish, Mandarin all express ideas, but each has its own grammar. Python, SQL, and JavaScript are all programming languages, each with its own syntax for different jobs.

Python was designed to read like plain English. You don't need semicolons, curly braces, or special symbols to end a line — which makes it the most beginner-friendly language for data work.

How Python compares to tools you already use:

Tool What it does well Where Python goes further
Excel Formulas, pivot tables, charts Automates multi-step workflows across many files
SQL Querying and filtering data in databases Adds logic, loops, and file handling around queries
Python Automation, data analysis, scripting End-to-end workflows that no single tool can match alone

You don't need to learn every programming language. Python is the right starting point for data and automation — it works equally well alongside SQL and Excel rather than replacing them.

Decision Flow

A visual walkthrough of the logic and process

Think of your code as a recipe your computer follows. By default, Python reads each line from top to bottom and executes it in order — this is called sequential execution. Understanding how that flow works (and how you can control it) is one of the most important ideas in programming.

Sequential Execution — the Default

When you write a Python program, each line is a single instruction. Python starts at the top and works its way down, running one instruction at a time. No lines are skipped, and no lines run out of order — unless you write code that tells Python to do something different.

Here is a simple three-step program. Notice how the output appears in exactly the same order as the lines in the code:

sequence.py 
print("Step 1: Collect the data")    # runs first
print("Step 2: Clean the data")      # runs second
print("Step 3: Summarise the data")  # runs last
Terminal $ python sequence.py
Step 1: Collect the data
Step 2: Clean the data
Step 3: Summarise the data

The Three Ways to Control Program Flow

Sequential execution is just the starting point. In later lessons you will learn three tools that let you shape how Python moves through your code:

Conditions

The fork in the road

Use if and else to make decisions. Your program takes one path when a condition is true, and a different path when it is false — like choosing a different route depending on traffic.

Loops

The repeat button

Use for and while to repeat a block of code. Instead of writing the same instruction ten times, you write it once and tell Python to loop — like stirring a sauce until it thickens.

Functions

The reusable shortcut

Give a block of instructions a name using def. Then call that name whenever you need those steps. It is like saving a recipe card — write it once, use it any time.

For now, just remember that Python always starts at the top and works downward. Every flow-control concept you learn in this module is simply a way to shape that path — skipping parts, repeating parts, or packaging parts into a reusable name.

Code Examples

Hands-on code snippets to explore the concepts

01

Member Greeting

Beginner Members print() Sequential Flow

What This Does

This is the simplest kind of Python program — three instructions that run one after another, from top to bottom. Imagine your system needs to display a summary card when a member calls in: their name, their health plan, and their member ID. The print() function tells Python to display each piece of text in the terminal. Because there is nothing to skip or repeat, the output always appears in exactly the same order as the code.

member_greeting.py 
print("Member: Jane Smith")       # instruction 1 — runs first
print("Plan: Gold PPO")           # instruction 2 — runs second
print("Member ID: MBR-00142")     # instruction 3 — runs last
Terminal $ python member_greeting.py
Member: Jane Smith
Plan: Gold PPO
Member ID: MBR-00142

Notice that the member's details appear in exactly the same order as the three print() lines. This is sequential execution — and it is the same pattern used whenever a system needs to display a member summary, a policy card, or a call-centre screen record.

02

Claim Cost

Beginner Claims Variables Arithmetic

What This Does

When a claim is submitted, the plan needs to work out how much the member owes and how much the plan will cover. This program uses variables — named containers that hold a value — to store the claim amount and the member's deductible, then calculates each party's share using simple arithmetic. Python works out the maths on the right side of the = sign and stores the result in a new variable before printing it.

claim_cost.py 
claim_amount = 350   # total amount billed by the provider
deductible   = 100   # amount the member must pay first

member_pays = deductible                   # member covers the deductible
plan_pays   = claim_amount - deductible    # plan covers the remaining balance

print("Member pays:", member_pays)   # display the member's share
print("Plan pays:",   plan_pays)     # display the plan's share
Terminal $ python claim_cost.py
Member pays: 100
Plan pays: 250

Every claims adjudication system does this kind of maths automatically. Python lets you express the same logic in plain, readable code — no formulas, no cell references, just variable names that describe exactly what the number represents.

03

Provider Claims

Beginner Providers Lists sum()

What This Does

Providers submit multiple claims throughout the month, and the plan needs to track how much has been billed in total. This program stores several claim amounts in a list — a single variable that holds a collection of items in order, similar to a column of numbers in a spreadsheet. The built-in sum() function adds every claim together automatically, so you do not need to add them manually one by one.

provider_claims.py 
claim_amounts = [120, 350, 85, 210]   # four claims submitted by one provider

total_billed = sum(claim_amounts)     # sum() adds every claim amount together

print(total_billed)                   # display the provider's total billed amount
Terminal $ python provider_claims.py
765

In the real world, a provider might submit hundreds of claims a month. Python's sum() works just as fast on a list of 10,000 claims as it does on four — making it a powerful tool for provider performance reporting and claims reconciliation.

02

Calculate a Copay

Beginner Claims Arithmetic print()

Your Task

A member submitted a claim for $400. Their plan requires a 20% copay on every claim. Write a Python program that stores the claim amount and the copay rate as variables, calculates how much the member owes, and prints the result.

copay_calc.py 
claim_amount = 400              # total amount on the claim
copay_rate = 0.20               # member pays 20% of the claim
member_owes = claim_amount * copay_rate  # calculate what's due
print("Member owes:", member_owes)       # display the result
Terminal $ python copay_calc.py
Member owes: 80.0

Python evaluates the right side of the = first, stores the result in member_owes, and then uses that value when printing — this is the same sequence a claims system follows every time it calculates cost-sharing.

03

Total Claim Amounts

Beginner Providers Arithmetic print()

Your Task

A provider submitted three separate claims: $120, $350, and $85. Write a Python program that stores each claim amount in its own variable, adds them together to get the total billed, and prints the final total.

provider_total.py 
claim_1 = 120                           # first claim amount
claim_2 = 350                           # second claim amount
claim_3 = 85                            # third claim amount
total_billed = claim_1 + claim_2 + claim_3   # sum all three
print("Total billed:", total_billed)         # display the total
Terminal $ python provider_total.py
Total billed: 555

Storing each claim in its own variable makes the program easy to read and update — if a claim amount changes, you only need to edit one number and the total recalculates automatically.

Common Mistakes

Pitfalls beginners hit when writing their first Python programs

Writing Instructions in Plain English Instead of Python Syntax

Python only reads precise, structured code — natural language phrases cause a SyntaxError.

Pitfall

When you are new to programming, it is natural to write instructions the way you would say them out loud. But Python doesn't understand English sentences — it only reads precise, structured code. If you type a plain English phrase, Python will throw a SyntaxError because it cannot parse the words as valid code.

Wrong — plain English 

Calculate 20% copay for the member

Correct — Python syntax 

copay = claim_amount * 0.20

Think of Python as a very literal assistant — it does exactly what you type, nothing more. Every instruction must follow exact syntax rules, not plain-English phrasing.

Assigning a Value Without Displaying It

Storing a value in a variable doesn't automatically show it on screen — you need print().

Pitfall

You might expect that writing copay = 80.0 will display 80.0 in your terminal. It won't — the value is stored silently in memory, and your program runs without showing anything. Python only displays output when you explicitly ask it to with print().

Wrong — no output shown 

claim_amount = 400
copay = claim_amount * 0.20
# Nothing appears on screen

Correct — use print() 

claim_amount = 400
copay = claim_amount * 0.20
print(copay)  # Displays: 80.0

Think of print() as the "publish" button — the variable is calculated silently in the background, and print() is what sends the result to your screen.

Calculating Without Storing the Result

If you don't assign the result to a variable, Python performs the calculation and immediately discards it.

Pitfall

When you write claim_amount * 0.20 on its own, Python runs the multiplication but immediately throws away the answer. Without a variable to hold the result, you can't use that value anywhere else in your program — and because Python won't report an error, this mistake is easy to miss.

Wrong — result discarded 

claim_amount = 400
claim_amount * 0.20  # Calculated, then lost

Correct — assign to a variable 

claim_amount = 400
copay = claim_amount * 0.20  # Saved for later

Every result you'll need later must be saved to a variable. Without one, the answer disappears as soon as that line finishes running — like writing on a whiteboard and immediately erasing it.

Real-World Use

How Python programs are used across healthcare insurance workflows

In healthcare insurance, analysts write Python programs every day to process data, catch errors, and produce the reports that teams rely on. Each program is just a sequence of instructions — the same concept you've been learning — applied to real data like member records, claim transactions, and provider information.

Members

  • Eligibility checks — verify whether a member is active before processing a claim or referral
  • Enrollment reports — count how many members joined, renewed, or lapsed in a given month
  • Plan assignment — match each member to the correct plan tier (Bronze, Silver, Gold) based on their record

Claims

  • Copay calculation — compute how much a member owes based on their plan's cost-sharing rules
  • Duplicate detection — flag claim records that appear more than once to prevent overpayment
  • Spend summaries — total claim amounts by member, provider, or date range for monthly reporting

Providers

  • Network validation — check whether a provider is in-network before a claim is approved
  • Billing totals — sum all claims billed by a provider over a period to spot outliers or billing errors
  • Credential checks — verify that a provider's licence and specialty are current before processing referrals
