Overview

A high-level summary of the topic

Py

A loop is Python's way of saying "do this for every item" — one instruction that runs automatically across an entire list, so you never copy and paste code again.

Think of a for loop like a factory conveyor belt: the belt moves each item past the same workstation, performs the same action on it, and then advances to the next — automatically, until every item on the belt has been processed.

The Loop

The conveyor belt — moves items forward one by one

A loop repeats the same block of code for every item in a sequence, so the work gets done without you writing duplicate instructions.

The for keyword

The start switch — sets the belt in motion

Writing for item in collection tells Python which list to use and what to call each item as it passes through the loop body.

range()

The item counter — loads exactly N items onto the belt

When you need to repeat something a set number of times, range() generates that many numbers for the loop to work through.

The loop body

The workstation — performs the same task each time

The indented code inside the loop runs once for every item on the belt — whether that's printing, calculating, or storing a result.

In healthcare analytics, a single for loop can process every claim in a file, every member in an eligibility export, or every provider in a billing report — tasks that would take hours to do manually in Excel.

Key Concepts

Loop, for loop, range(), and the accumulator pattern — with healthcare examples.

Loop

A repeated block of code

loop

A loop is a Python instruction that repeats a block of code automatically — once for each item in a list, or for a set number of times. Instead of writing the same line ten times, you write it once and tell Python to repeat it.

Manual vs Loop

Manual Loop
Example print("Alice") for m in members:
For 100 items 100 separate lines Same single loop
Adding an item Edit code each time Update the list only
Python 
member_names = ["Alice", "Bob", "Carol"]  # list of three members

for member in member_names:               # loop runs once per member
    print(member)                         # prints current member name

# Output: Alice  Bob  Carol (one per line)

Loops scale automatically: add a fourth member to member_names and the loop processes it without any change to the loop code itself.

for Loop

Visits each item in sequence

for

A for loop moves through each item in a collection one at a time, running its indented code block once per item. The variable you name after for automatically holds the value of the current item on each pass.

for Loop Anatomy

Start with for for ✓
Name the loop variable clearly member amount ✓ x avoid
Use in before the list in members ✓
End the line with a colon : in members: ✓ in members ✗
Indent the loop body 4 spaces print(member) ✓
Python 
plans = ["HMO", "PPO", "EPO"]  # three insurance plan types

for plan in plans:              # 'plan' holds each value in turn
    print(plan)                 # prints: HMO, then PPO, then EPO

# Python stops automatically after EPO

Name the loop variable to match what each item represents: use for member in members rather than for x in members — readable names make the logic obvious at a glance.

range()

Generates a number sequence

range

The range() function generates a sequence of integers that a for loop can iterate through. The stop number is always excluded — range(1, 6) produces 1, 2, 3, 4, 5 (not 6).

range() Forms

Form Meaning Produces
range(5) 0 up to stop 0, 1, 2, 3, 4
range(1, 6) Custom start 1, 2, 3, 4, 5
range(0, 10, 2) Custom step 0, 2, 4, 6, 8
Python 
for day in range(1, 6):              # generates 1, 2, 3, 4, 5 (not 6)
    print(f"Day {day}: visit logged") # runs exactly 5 times

for i in range(5):                   # generates 0, 1, 2, 3, 4
    print(i)                         # starts at 0 by default

The stop value is never included: range(1, 6) ends at 5, not 6. This is the same rule Python uses for list slices — the end index is always exclusive.

Accumulator

Building a running total

pattern

An accumulator is a variable set to zero before a loop that collects values as the loop runs. Each iteration adds the current item to the running total, so when the loop finishes, the variable holds the final sum. It is the most common pattern for calculating totals from a dataset.

Accumulator Pattern — 3 Steps

1 — Initialise before the loop total = 0 ✓ total = 0 inside loop ✗
2 — Add each value inside the loop total = total + amount ✓
3 — Read the total after the loop print(total) outside loop ✓
Python 
claim_amounts = [480.00, 1250.00, 375.00]  # three claim values

total = 0                                   # step 1: initialise before loop

for amount in claim_amounts:               # step 2: visit each claim
    total = total + amount                 # add current claim to running total

print(total)                               # step 3: read total — prints 2105.0

Always initialise the accumulator before the loop: if you write total = 0 inside the loop body it resets to zero on every iteration, and you will only ever see the last value added — not the true sum.

while Loop

Repeats while a condition is True

while

A while loop keeps repeating its code block for as long as a condition stays True. Unlike a for loop — which knows exactly how many items to visit — a while loop checks its condition before every iteration and stops only when that condition becomes False.

for vs while

for while
Use when You know the items You check a condition
Iteration count Fixed / known Unknown upfront
Common risk Off-by-one in range Infinite loop if condition never False
Python 
retries = 0                            # track how many attempts
max_retries = 3                        # stop after 3 tries

while retries < max_retries:           # condition checked before every loop
    print(f"Attempt {retries + 1}")   # print current attempt number
    retries = retries + 1              # update variable or loop runs forever

print("Done")                          # runs after the loop finishes

Always update the variable the condition checks: if retries never increases, the condition retries < 3 stays True forever — causing an infinite loop that freezes your program.

Nested Loops

A loop inside another loop

nested

A nested loop is a loop placed inside another loop's indented code block. The inner loop runs completely from start to finish for every single iteration of the outer loop. This is how you process combinations — every member paired with every plan, or every row paired with every column.

How Nested Loops Work

Outer loop runs once per member for member in members:
Inner loop runs completely for each member for plan in plans:
Total iterations = outer × inner 3 members × 2 plans = 6 lines
Inner body indented 8 spaces total print(...) ✓ print(...) ✗
Python 
members = ["Alice", "Bob", "Carol"]  # outer list — 3 members
plans   = ["HMO", "PPO"]             # inner list — 2 plan types

for member in members:               # outer loop: runs 3 times
    for plan in plans:               # inner loop: runs 2 times per member
        print(f"{member} — {plan}") # prints all 6 member-plan combinations

# Output: Alice — HMO, Alice — PPO, Bob — HMO, Bob — PPO …

Total iterations multiply, not add: a nested loop over 100 members and 50 plans runs 100 × 50 = 5,000 iterations. Keep inner loops short, and consider whether a single loop with the right data structure is simpler.

Which Loop Should You Use?

Python gives you two types of loop. The rule of thumb is simple: reach for for when you have a collection, and reach for while when you have a condition.

for Loop

Known list or range

Use a for loop when you have a collection of items or a fixed count — such as every member in a list, every row in a file, or days 1 to 30.

while Loop

Unknown count, condition-based

Use a while loop when you want to repeat until a condition changes — such as retrying until a record uploads, or looping until a total exceeds a threshold.

Nested Loops

Combinations and cross-products

Place a for loop inside another when you need every item in one list paired with every item in another — such as every member with every insurance plan.

When your loop produces unexpected output, trace through it using the table above: write down what the loop variable holds on each pass and what the body code would print. This single habit solves the majority of loop bugs before you ever need a debugger.

01

Flag Denied Claims

Beginner Claims counter pattern if inside loop

What This Does

This script loops through a list of claim statuses and uses a counter variable to count how many were denied. On each pass, an if check decides whether to increment the counter.

flag_denied_claims.py 
claim_statuses = ["Approved", "Denied", "Approved", "Pending", "Denied"]  # five claim outcomes

denial_count = 0                          # start the counter at zero — no denials yet

for status in claim_statuses:             # loop through each claim status in the list
    if status == "Denied":               # check if this particular status is a denial
        denial_count = denial_count + 1   # yes — add 1 to the running count

print("Total denied claims:", denial_count)  # print the final tally after all claims
Terminal $ python flag_denied_claims.py

Change "Denied" to "Pending" on the condition line and the same loop counts pending claims instead — no other code changes needed.

02

Deductible Remaining

Beginner Members accumulator subtraction

What This Does

This script loops through the payments a member has already applied towards their deductible, accumulates the total paid, then subtracts it from the annual deductible to show what is still owed.

deductible_remaining.py 
annual_deductible = 1500.00                     # the member's full yearly deductible

payments_applied = [200.00, 450.00, 375.00]     # amounts already credited to the deductible

total_applied = 0                               # accumulator — starts at zero before the loop

for payment in payments_applied:                # loop through each credited payment
    total_applied = total_applied + payment     # add this payment to the running total

remaining = annual_deductible - total_applied   # subtract total paid from the annual amount
print("Deductible remaining: $" + str(remaining))  # print how much is still owed

03

Highest Billed Provider

Beginner Providers max tracking if inside loop

What This Does

This script loops through a list of providers and their billed amounts, tracking the highest amount seen on each pass. When the loop finishes, it prints which provider billed the most.

highest_billed_provider.py 
provider_data = [                           # list of [provider name, billed amount] pairs
    ["Dr. Singh", 3200.00],
    ["Eastside Radiology", 6750.00],
    ["Metro Urgent Care", 1480.00],
]

top_provider = ""   # will hold the winning provider name — empty until loop runs
top_amount   = 0    # will hold the highest amount seen — starts at zero

for entry in provider_data:              # loop through each [name, amount] pair
    name   = entry[0]                   # first item in the pair is the provider name
    amount = entry[1]                   # second item is the billed amount
    if amount > top_amount:             # is this amount bigger than the current top?
        top_provider = name             # yes — this provider takes the top spot
        top_amount   = amount           # update the highest amount seen so far

print(top_provider + " — $" + str(top_amount))  # print the highest-billing provider
Terminal $ python highest_billed_provider.py
Eastside Radiology — $6750.0

This max-tracking pattern works for any dataset of any size — add 50 more providers to the list and the same loop finds the highest biller without any changes to the logic.

Member Copay Report

Your Task

Your clinic has two parallel lists — one with member names and one with the copay amount each member owes. Write a loop that uses range(len()) to step through both lists by index and prints one formatted line per member showing their name and copay. Your output should show three lines, one for each member.

member_names = ["David Park", "Sarah Nguyen", "James Okafor"]  # member name list

Common Mistakes

Frequent errors and how to avoid them

Loop Body Not Indented — Python Raises an IndentationError

Python uses indentation to identify the loop body — a statement with no indent causes an immediate IndentationError.

Pitfall

Every line that belongs inside a loop must be indented by exactly 4 spaces — print(status) with no indent raises an IndentationError because Python can't find the loop body. Add 4 spaces before each statement in the loop body to fix it.

Wrong — no indent — IndentationError 

claim_statuses = ["Approved", "Pending", "Denied"]

for status in claim_statuses:
print(status)    # not indented — IndentationError

Correct — 4-space indent — runs correctly 

claim_statuses = ["Approved", "Pending", "Denied"]

for status in claim_statuses:
    print(status)    # indented 4 spaces — Python recognises this as the loop body

Think of indentation as the folder that holds everything the loop owns — any statement that isn't indented runs after the loop ends, not inside it, so Python never treats it as part of the loop.

No Colon After the for Statement — Python Raises a SyntaxError

The colon at the end of a for statement is required — omitting it raises a SyntaxError before the loop can run at all.

Pitfall

Python's for line must always end with a colon — for member_id in member_ids with no colon raises SyntaxError: expected ':' immediately. Add the colon and Python knows the indented block below is the loop body.

Wrong — no colon — SyntaxError 

member_ids = [1042, 1087, 1113]

for member_id in member_ids      # missing colon — SyntaxError
    print(member_id)

Correct — colon present — runs correctly 

member_ids = [1042, 1087, 1113]  # member ID list to process

for member_id in member_ids:     # colon signals the start of the loop body
    print(member_id)             # this line runs once per ID in the list

The colon-and-indent pattern is the same for every Python block — if, for, while, and functions all follow it, so building this habit now pays off across every concept you learn next.

Initialising the Accumulator Inside the Loop — Total Resets on Every Pass

Starting the running total inside the loop body resets it to zero on each pass, so the final result holds only the last item's value — not the sum of all items.

Pitfall

Placing total_copay = 0 inside the loop body resets the accumulator back to zero on every pass — there is no crash, but the final total only reflects the last item instead of the sum of all items. Move the initialisation to the line before the loop so the total survives every pass.

Wrong — reset every pass — wrong total 

copay_amounts = [25.00, 40.00, 15.00]

for copay in copay_amounts:
    total_copay = 0              # resets to zero on every pass
    total_copay = total_copay + copay

print(total_copay)               # prints 15.0 — only the last copay

Correct — initialised before loop — correct sum 

copay_amounts = [25.00, 40.00, 15.00]  # three member copay amounts

total_copay = 0                        # initialise once — before the loop starts

for copay in copay_amounts:
    total_copay = total_copay + copay  # accumulates across every pass

print(total_copay)                     # prints 80.0 — the correct total

Think of a cashier who clears their till once before the first customer — not before every customer — the accumulator works the same way: one reset before the loop starts, then additions all the way through, which is exactly how a claims system builds a running monthly total.

Real-World Use

How loops are used across healthcare insurance workflows

In healthcare insurance, loops allow analysts to process entire lists of member records, claim transactions, and provider information with a single block of code instead of writing one instruction per row. Every repeated task — from summing copay amounts across a billing period to checking each claim's status before adjudication — becomes one loop that scales to any dataset size without changing the code.

Members

  • Eligibility batch checks — a for loop steps through every member ID in an enrollment export and checks whether coverage is active.
  • Copay accumulation — an accumulator loop adds each member's copay_amount to a running total to produce a monthly cost summary.
  • Outreach list formatting — a loop formats each member's name and plan tier into a personalised row for a bulk outreach file.

Claims

  • Total billed calculation — a for loop accumulates each claim_amount into a single total_billed variable across a full billing cycle.
  • Status queue counting — a loop checks each claim's status field and increments a counter whenever it equals "Pending" so the queue depth is known before adjudication.
  • High-cost flagging — a while loop works through a prioritised queue and flags any claim_amount above the high-cost threshold for manual review.

Providers

  • Fee schedule comparison — a loop compares each provider's billed_amount against the allowed fee from the schedule and prints any amount that exceeds the limit.
  • Auth step sequencing — a while loop steps through each prior-authorisation requirement in order until every step has been confirmed.
  • Network validation — a loop checks each provider's network_status in a referral list and collects only the in-network names for the referral report.
