Lesson Objective

The goal and expected outcome of this lesson

What a variable is

A variable is a named label you attach to a value so Python can find it later.

Storing values in Python

Python saves a value in memory and links it to whatever variable name you choose.

Python's core data types

Python uses data types to tell the interpreter whether a value is a number, text, or true/false.

Checking a value's type

Python's built-in type() function returns the data type of any variable or value you give it.

This lesson introduces variables and data types, which are the building blocks behind every Python program you will write.

Overview

A high-level summary of the topic

Py

Every Python program stores information in variables — named containers that hold a specific type of value your code can read and update at any point.

Think of a filing cabinet where every drawer has a label on the front. The label is the variable name, the document inside is the value, and the type of document tells you what you can do with it. This lesson covers the four most common document types you will use in every Python script.

Variable

The drawer — holds one value by name

The label on the drawer — gives any value a name your script can refer back to.

String

The text doc — stores words and sentences

Text documents in the cabinet — anything you read rather than calculate.

int & float

The number sheet — counts or measures

Number sheets in the cabinet — whole counts or decimals, depending on your data.

Boolean

The checkbox — only True or False

The checkbox on the drawer — ticked or unticked, nothing in between.

Why this matters for your work: every data script you write will store, transform, and compare values. Getting comfortable with these four types early means you'll spend less time debugging type errors and more time analysing results.

Key Takeaways

The most important ideas to remember

Variables Eliminate Repetition

Instead of typing the same value in ten places, you store it once in a variable and Python reuses that label everywhere — so a single update fixes every reference at once.

DRY principle reusability single source

Data Type Controls Behaviour

Python uses a value's type to decide what operations are allowed — adding two integers gives a number, but adding two strings joins them as text, which is why "3" + "4" returns "34", not 7.

type rules concatenation arithmetic

type() Reveals Hidden Types

When a script pulls data from a CSV file or SQL query, values often arrive as the wrong type — for example a price stored as text — and calling type() is the fastest way to diagnose and fix that before your calculations break.

debugging type() data quality

Key Concepts

Core terms and definitions for this topic

Variable

Named storage for a value

var

A variable is a named label that stores a value in memory. Create one with a name, an equals sign, and the value — Python handles the rest. You can reassign it at any time.

Naming Rules

Start with a letter or _ name _total ✓
Letters, numbers, _ only user_id_2 ✓
No spaces or symbols total price ✗ → total_price
Lowercase + underscores first_name over FirstName
Python 
city = "Los Angeles"  # store a text value
print(city)           # Los Angeles

price = 20
quantity = 3
total = price * quantity  # 20 × 3 = 60
print(total)              # 60

Python keyword alert: Names like print or if are reserved — you can't use them as variable names. Use descriptive names like total_price.

String

Text wrapped in quotes

str

A string is any text wrapped in quotation marks — letters, digits, spaces, or punctuation. The quotes tell Python to treat the content as text, not as code to run.

Quote Styles

'hello' Single quotes
=
"hello" Double quotes
→
Both work!
Python 
first_name = "Alice"        # double quotes
city = 'Sydney'              # single quotes — both work
employee_id = "EMP-00142"  # digits in quotes = still a string
greeting = "Hello, " + first_name  # join strings with +
print(greeting)             # Hello, Alice

Watch out: "3" + "4" gives "34" (joined text), not 7. Remove the quotes when you need arithmetic.

Integer

Whole number, no decimal

int

An integer (int) is a whole number — no decimal point. Use it to count things: rows in a dataset, a customer's ID, or how many times a loop runs.

Python 
orders = 25
items_per_order = 4
total_items = orders * items_per_order  # 100
leftover = total_items % 3              # remainder = 1
print(total_items, leftover)            # 100 1

Arithmetic Operators

Op Meaning Example
+ Add 3 + 4 → 7
- Subtract 10 - 6 → 4
* Multiply 5 * 3 → 15
// Integer divide 7 // 2 → 3
% Remainder 7 % 2 → 1

CSV data arrives as text: use int("42") to convert a string into a real integer before arithmetic — otherwise Python raises a TypeError.

Float

Number with a decimal point

float

A float is any number with a decimal point. Even 5.0 is a float while 5 without the dot is an integer. Use floats for prices, rates, and averages.

int vs float

int float
Example 5 5.0
Has decimal ✗ No ✓ Yes
Use for Counting Prices, rates
Python 
price = 19.99
discount = 0.15   # 15%
tax_rate = 0.08   # 8%
final_price = price * (1 - discount) * (1 + tax_rate)
print(round(final_price, 2))  # 18.33
print(type(final_price))      # <class 'float'>

int × float = float: 10 * 1.5 returns 15.0. Convert back with int(15.0) to get 15.

Boolean

True or False — that's it

bool

A boolean holds exactly True or False — Python's built-in yes/no answer. Always capitalised — lowercase true will cause a NameError.

True
is_active = True

Condition is met / yes

False
is_active = False

Condition not met / no

Python 
score = 85
passed = score >= 60   # is 85 >= 60? → True
print(passed)          # True

on_sale = score < 50   # is 85 < 50? → False
print(on_sale)         # False

Booleans power if-statements: if is_active: checks whether the value is True and runs the block only if it is. You'll see this pattern from Lesson 06 onwards.

Member Profile

This script stores four pieces of member information as variables — one for each data type — then prints them. Each variable holds a different kind of value: text, a whole number, a decimal, or true/false.

02

Claim Cost Split

Beginner Claims float arithmetic derived variable

What This Does

This script stores the total claim amount and the member's deductible, then calculates what each party owes. It shows that a variable can hold the result of a calculation, not just a value you type in.

claim_cost_split.py 
claim_amount         = 850.00  # total billed for the procedure
deductible_remaining = 200.00  # amount still owed on the member's deductible

member_owes = deductible_remaining        # member pays the deductible balance first
plan_covers = claim_amount - member_owes  # plan pays everything above the deductible

print("Total claim:", claim_amount)  # show the full billed amount
print("Member owes:", member_owes)    # show the member's share
print("Plan covers:", plan_covers)    # show the plan's share
Terminal $ python claim_cost_split.py
Total claim: 850.0
Member owes: 200.0
Plan covers: 650.0

You can reuse this script for any claim by changing just the two numbers at the top. Python recalculates everything below automatically.

Provider Billing Check

Beginner Providers type() data validation

This script stores four provider fields and uses Python's built-in type() function to check what kind of value each variable holds. It's a quick way to catch mistakes before they cause errors in your calculations.

02

Split a Claim

Beginner Claims float, Arithmetic

Your Task

A member filed a claim for $620. Their plan covers 80% of every claim — the member pays the remaining 20%. Store the total claim and the plan's coverage rate as float variables, then calculate what the plan pays and what the member owes. Print all three amounts.

claim_split.py 
claim_total = 620.00          # float — total amount on the claim
plan_coverage_rate = 0.80     # float — plan pays 80% of every claim

plan_pays = claim_total * plan_coverage_rate           # calculate plan share
member_owes = claim_total * (1 - plan_coverage_rate)  # calculate member share

print("Claim total:", claim_total)    # print total
print("Plan pays:", plan_pays)        # print plan amount
print("Member owes:", member_owes)    # print member amount

Common Mistakes

Frequent errors and how to avoid them

Storing a Member Name Without Quotation Marks

Without quotes, Python treats the value as a variable name and raises a NameError.

Pitfall

Python uses quotation marks to distinguish text from a variable name — without them, member_name = Sofia Reyes makes Python look for a variable called Sofia, which doesn't exist, so it raises a NameError. Wrap any text value in single or double quotes to fix it.

Wrong — no quotes 

member_name = Sofia Reyes  # NameError

Correct — quoted string 

member_name = "Sofia Reyes"  # str — quotes mark this as text

Think of quotes as the label holder on a filing folder — the text inside is the member's name, not an instruction for Python to go and find something.

Storing a Claim Amount Inside Quotation Marks

Wrapping a number in quotes makes it a string, so Python raises a TypeError when you try to do arithmetic with it.

Pitfall

Putting quotes around a number — claim_amount = "620.00" — stores it as text, so claim_amount * 0.20 raises a TypeError because Python can't multiply text by a number. Remove the quotes from any value you need to calculate with.

Wrong — number in quotes 

claim_amount = "620.00"
copay = claim_amount * 0.20  # TypeError

Correct — float without quotes 

claim_amount = 620.00         # float — no quotes means a real number
copay = claim_amount * 0.20  # Works: 124.0

A simple rule: if you will ever add, subtract, multiply, or divide the value, store it without quotes. Think of the difference between writing "$620" on a sticky note (text) versus entering 620 into a spreadsheet cell (a real number you can calculate with).

Writing true or false With a Lowercase First Letter

Python booleans must be capitalised exactly — lowercase true or false raises a NameError.

Pitfall

Python's boolean keywords are exact spellings: True and False with a capital first letter — writing in_network = true in lowercase makes Python look for a variable named true, which doesn't exist, and raises a NameError.

Wrong — lowercase bool 

in_network = true   # NameError: name 'true' is not defined

Correct — capitalised keyword 

in_network = True   # bool — capital T is the Python keyword

You only need to remember two exact spellings: True and False. Most code editors highlight them in a distinct colour — if your boolean does not change colour after you type it, that is your first clue that the capitalisation is wrong.

Real-World Use

How variables and data types are used across healthcare insurance workflows

In healthcare insurance, every script starts by naming its data — a member's ID, a claim total, a provider's network status. Variables and data types are what make those values reusable and calculable throughout the program.

Members

  • Member record storage — Name, member ID, premium, and active status each get their own variable.
  • Premium calculations — A float variable holds the monthly premium so it can be multiplied, compared, or updated in one place.
  • Eligibility flags — A bool like is_active records whether a member's coverage is current.

Claims

  • Billed amount storage — A float variable holds the claim total so copay and plan-share can be calculated from it.
  • Cost-share splitting — plan_coverage_rate and claim_total combine in one line to split what the plan pays versus what the member owes.
  • Claim ID labelling — An int variable stores the claim's unique identifier for logging and comparison.

Providers

  • Provider identification — An int holds the NPI number and a str holds the name, ready for matching and display.
  • Network status flag — in_network = True or False routes the claim to the correct reimbursement rate.
  • Billed totals — A float variable stores the amount billed so it can be compared against fee schedules or summed across claims.
