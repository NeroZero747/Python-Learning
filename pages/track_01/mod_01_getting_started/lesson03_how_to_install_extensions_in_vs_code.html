Overview

A high-level summary of the topic

Ext

VS Code is a plain text editor out of the box — but with the right extensions installed, it becomes a full Python development environment that catches your mistakes before you even run your code.

VS Code bare is like a word processor with no spell-check — you can type, but it won’t tell you when you’ve made a mistake. Extensions add the smarts: error highlighting, auto-complete, debugging, and formatting, all in one place. Here are the four you need for Python:

Python (by Microsoft)

The engine — makes Python development work

Run scripts, debug code, and get IntelliSense completions. Without this, VS Code doesn’t understand Python at all.

Pylance

The analyst — catches issues as you type

Advanced type-checking, faster code analysis, and rich auto-complete suggestions that understand your entire codebase.

Jupyter

The notebook — run code cell by cell

Open and run .ipynb notebook files directly in VS Code — essential for data analysis and exploration workflows.

Black Formatter

The cleaner — formats your code automatically

Auto-formats your Python code to the Black style standard on every save — consistent, readable code with zero effort.

With these four extensions, VS Code gives you:

Syntax highlighting IntelliSense completions Step-through debugger Notebook support Auto-formatting

Extensions are free, installed in seconds through the VS Code Extensions panel, and can be enabled or disabled at any time. You don’t need to install hundreds — start with these four and your environment is production-ready.

Key Concepts

Core terms and definitions for this topic

What is an Extension?

Definition

A VS Code extension is a small software package that adds new features to the editor. Extensions are installed from the VS Code Marketplace — for free — and activate automatically based on the file type you open.

Without extensions, VS Code is just a text editor. With them, it becomes a full IDE. The extensions below are the ones every Python developer installs:

Extension Purpose Priority
Python Run, debug, IntelliSense Required
Pylance Type checking & analysis Required
Jupyter Notebook support Recommended
Black Formatter Auto-format on save Recommended

Open the Extensions panel with Ctrl + Shift + X (Windows) or Cmd + Shift + X (Mac), then search for each extension by name.

Python Extension (by Microsoft)

Definition

The Python extension (published by Microsoft) is the core extension for Python development in VS Code. It connects VS Code to whichever Python interpreter is installed on your machine and enables the full development workflow.

When you open a .py file, the Python extension activates automatically and unlocks:

Run Python scripts
Step-through debugger
IntelliSense completions
Interpreter selection
Jupyter Notebook integration

Always install the one by Microsoft — search for "Python" in the Extensions panel and look for the publisher badge that says "Microsoft".

Pylance

Definition

Pylance is a language server for Python that works alongside the Python extension. It gives VS Code a deeper understanding of your code — not just whether it runs, but whether it’s correct.

Here’s what Pylance catches that the basic Python extension misses:

script.py — Pylance catches this error before you run 
def greet(name: str) -> str:
    return "Hello, " + name

result = greet(42)   # Pylance: Argument of type "int"
                     # is not assignable to parameter "str"

Pylance is usually installed automatically with the Python extension, but you can also install it separately. Set python.analysis.typeCheckingMode to "basic" or "strict" in settings.

How to Install an Extension

A 3-step process that works for any extension

1

Open the Extensions panel

Press Ctrl + Shift + X (Windows) or Cmd + Shift + X (Mac), or click the blocks icon in the left sidebar.

2

Search for the extension by name

Type the extension name in the search box. Check that the publisher matches (e.g., Microsoft for Python and Pylance).

3

Click Install, then Reload if prompted

Some extensions need VS Code to reload before activating. Click Reload Window if the prompt appears.
