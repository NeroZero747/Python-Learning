This lesson completes the data pipeline — turning a processed, clean DataFrame back into a shareable, deliverable file.

Overview

A high-level summary of the topic

Py

Exporting sends your cleaned, transformed DataFrame back to a file — a CSV anyone can open or an Excel workbook with multiple sheets.

Think of exporting like printing a finished report — your analysis is done, so you choose the format and deliver it to whoever needs it.

to_csv()

The plain-text print — saves every row as CSV

You call df.to_csv("file.csv", index=False) and pandas writes every row and column to a plain-text file that Excel, Google Sheets, or any database import tool can open immediately. It accepts optional parameters like sep=";" for European-style delimiters and encoding="utf-8-sig" to keep special characters intact when the file is opened in Excel.

to_excel()

The spreadsheet print — creates an xlsx file

You call df.to_excel("file.xlsx", index=False) and pandas creates a real .xlsx workbook with typed columns, so numbers stay as numbers and dates stay as dates. You need the openpyxl package installed, and you can pass sheet_name="Summary" to label the worksheet tab instead of the default "Sheet1".

ExcelWriter

The multi-sheet print — several tabs at once

You use pd.ExcelWriter("file.xlsx") as a context manager and call to_excel() multiple times inside it, each with a different sheet_name. This lets you bundle several DataFrames — for example revenue, costs, and headcount — into one workbook that stakeholders can navigate with sheet tabs.

index parameter

The row-number switch — include or exclude it

By default, both to_csv() and to_excel() write the DataFrame's numeric index as a first column — a column of 0, 1, 2, … that most recipients don't need. Setting index=False removes it, giving you a clean file with only data columns.

Exporting data is the final step that turns your analysis into something other people can act on — master these methods and you become the person who delivers clean, ready-to-use files.

Key Takeaways

The most important ideas to remember

Encoding Protects Special Characters

Setting encoding='utf-8-sig' when writing a CSV ensures Excel displays accented characters and currency symbols correctly. Without it, non-ASCII text appears as garbled symbols, which is a common complaint from international teams.

utf-8-sig encoding= Special chars

ExcelWriter Unlocks Multi-Sheet Workbooks

One script run can replace 30 minutes of manual sheet creation, pasting, and header formatting that analysts repeat every reporting cycle. Automating multi-sheet workbooks also eliminates copy-paste errors that creep in when you build board packs by hand.

ExcelWriter openpyxl Multi-sheet

Timestamp Filenames Prevent Silent Overwrites

Calling to_csv() on the same filename a second time silently replaces the old file with no undo. Adding the current date to the filename — report_2026-03-30.csv — creates an automatic archive trail so you can always recover yesterday's version.

Date stamp Versioning f-string

Exporting Fewer Columns Keeps Files Clean

Selecting only the columns your recipient needs with df[cols].to_csv() before exporting avoids sharing internal fields like IDs or raw scores. A 50-column dump overwhelms stakeholders — a 5-column extract gets read.

df[cols] columns= Subset

Key Concepts

to_csv(), to_excel(), index control, and column selection.

Exporting to CSV

Save a DataFrame as a CSV file

method

to_csv() writes a DataFrame to a comma-separated text file. Pass the output filename as a string.

Key Parameters

path Output filename (required)
index=False Exclude the row numbers
sep Delimiter — default ,
encoding Character set — default utf-8
Python 
df.to_csv("output.csv", index=False)   # save without row numbers
df.to_csv("euro.csv", sep=";")          # semicolon-separated
print("CSV saved")

Always pass index=False. Without it, Pandas adds a column of row numbers that clutters the output file.

Exporting to Excel

Save as an .xlsx workbook

method

to_excel() writes a DataFrame to an Excel file. Use sheet_name to name the sheet.

Key Parameters

path Output filename (required, .xlsx)
index=False Exclude row numbers
sheet_name Tab name — default Sheet1
float_format Number format — e.g. "%.2f"
Python 
df.to_excel("report.xlsx", index=False)  # save to Excel
df.to_excel("report.xlsx",
            sheet_name="Sales",           # custom sheet name
            index=False)

Install openpyxl first. Run pip install openpyxl — Pandas needs it to write .xlsx files.

Index Control

Include or exclude the index

parameter

The index is the row label column. By default, to_csv() and to_excel() include it. Set index=False to exclude it.

With vs Without Index

index=True index=False
Extra column ✓ 0, 1, 2… ✗ None
File size Slightly larger Slightly smaller
Best for Time-series index Most exports
Python 
df.to_csv("with_index.csv")             # includes 0, 1, 2…
df.to_csv("clean.csv", index=False)     # no extra column
print("Both files saved")

The extra column looks like unnamed data. Recipients of your file may not know what the index column means. Use index=False unless the index carries meaning.

Exporting Columns

Export only selected columns

technique

Exporting specific columns means selecting a subset of columns before calling the export method. This keeps the output file tidy and focused.

Column Selection Patterns

df[["A","B"]].to_csv(...) Select then export
usecols on re-import Load only needed columns
Store column list in a variable cols = ["A", "B"]
Python 
cols = ["Name", "Region", "Sales"]         # columns to export
df[cols].to_csv("subset.csv", index=False) # export only those
print(df[cols].shape)                       # confirm column count

Watch out for sensitive fields. Internal IDs, salary figures, or raw scores can slip into exports if you don't filter columns first. Always review the column list before sharing a file outside your team.

Multi-Sheet & Append

Write multiple sheets or append to an existing file

technique

ExcelWriter is a context manager that keeps a workbook open while you write multiple DataFrames to separate sheets. You can also open an existing file in mode='a' to add a new sheet without deleting the sheets already there.

Key Parameters

path Output .xlsx filename
mode='a' Append — adds sheets to an existing file
if_sheet_exists error (default), replace, or overlay
engine='openpyxl' Required for append mode
Python 
# Write two DataFrames to separate sheets in one file
with pd.ExcelWriter("board_pack.xlsx") as writer:
    revenue.to_excel(writer, sheet_name="Revenue", index=False)
    costs.to_excel(writer, sheet_name="Costs", index=False)

# Append a new sheet to an existing workbook
with pd.ExcelWriter("board_pack.xlsx",
                     mode="a",                    # append mode
                     engine="openpyxl",           # required for append
                     if_sheet_exists="error"      # prevent overwrite
                     ) as writer:
    headcount.to_excel(writer, sheet_name="People", index=False)

Always use the with statement. ExcelWriter needs to close the file properly to write valid Excel data. If you skip the context manager and the script crashes, the file may be corrupted.

Save to CSV

Beginner Reports to_csv() index=False

What This Does

This script saves a DataFrame to a CSV file. Setting index=False prevents pandas from adding an extra index column.

04

Multi-Sheet Excel

Intermediate ExcelWriter multi-sheet context manager

What This Does

This script uses ExcelWriter to write two DataFrames into separate sheets inside a single workbook. The with statement ensures the file is saved and closed automatically.

multi_sheet.py 
import pandas as pd  # load pandas

sales = pd.read_csv("sales.csv")       # first dataset
returns = pd.read_csv("returns.csv")   # second dataset

# Write both DataFrames to one workbook
with pd.ExcelWriter("report.xlsx") as writer:   # open workbook
    sales.to_excel(writer,                       # write first sheet
                   sheet_name="Sales",
                   index=False)
    returns.to_excel(writer,                     # write second sheet
                     sheet_name="Returns",
                     index=False)

print("Two sheets saved to report.xlsx")  # confirm
Terminal $ python multi_sheet.py
Two sheets saved to report.xlsx

Always use the with statement with ExcelWriter — it guarantees the workbook is saved and closed even if an error occurs midway.

05

Append to CSV

Intermediate mode="a" header=False append

What This Does

This script appends new rows to an existing CSV without overwriting it. Setting mode="a" opens the file in append mode, and header=False prevents writing the column names a second time.

append_csv.py 
import pandas as pd  # load pandas

new_rows = pd.read_csv("daily_log.csv")  # today's new data

# Append rows to the existing file (don't overwrite)
new_rows.to_csv("master_log.csv",
                mode="a",           # append, not overwrite
                header=False,        # skip column headers
                index=False)         # skip index column

print("Appended", len(new_rows), "rows")  # confirm
Terminal $ python append_csv.py
Appended 42 rows

If you forget header=False when appending, you will get a duplicate header row in the middle of your data that breaks downstream reads.

If you already use SQL SELECT … INTO OUTFILE or save a workbook from Excel, pandas to_csv() and to_excel() do the same thing — they write a DataFrame to a file on your drive.

SQL result sets have no row index, so this issue doesn't arise in SQL.

Row numbers are part of Excel's grid, not part of the data when saved.

Filters the DataFrame to a list of columns before writing so only those columns appear in the file.

Names the columns explicitly in the SELECT clause instead of using SELECT *.

Safe Daily Append

Intermediate mode="a" header=False

Your Task

Your daily pipeline produces a small CSV of new transactions. Append today's rows to the master transactions.csv without overwriting the existing data and without writing a duplicate header row.

daily_append.py 
import pandas as pd

today = pd.read_csv("today_transactions.csv")  # new rows

Common Mistakes

Pitfalls beginners hit when working with exporting data

Exporting With an Unwanted Index Column

The output file contains a nameless extra column of row numbers.

Pitfall

You open the exported CSV in Excel and notice an unnamed column of numbers (0, 1, 2 …) in column A. Recipients will ask "what is this?" because the index has no column header and no business meaning.

Wrong — index included 

df.to_csv("output.csv")   # adds 0, 1, 2… as first column

Correct — no index 

df.to_csv("output.csv", index=False)  # clean output
print("Saved without index column")    # confirm

Unless your index carries meaning (like dates), always add index=False. Recipients of the file rarely want a mystery column of numbers.

Overwriting an Existing File Without Warning

to_csv() silently replaces the old file with no confirmation prompt.

Pitfall

You run your script a second time and discover yesterday's export is gone — to_csv() replaced it without asking. There is no confirmation prompt and no recycle bin recovery, so the old data is permanently lost.

Wrong — overwrites old file 

df.to_csv("report.csv", index=False)  # old report is gone

Correct — timestamped name 

from datetime import date                        # date helper
filename = f"report_{date.today()}.csv"           # unique name
df.to_csv(filename, index=False)                   # safe export
print(f"Saved to {filename}")                      # confirm path

Adding today's date to the filename is like version-numbering a document. You can always go back to yesterday's export if something goes wrong.

Using to_csv() When You Need an Excel File

The recipient cannot open the .csv properly in Excel with formatting intact.

Pitfall

to_csv() creates plain text. If your recipient expects an .xlsx file with multiple sheets or formatting, use to_excel() instead.

Wrong — csv for Excel user 

df.to_csv("report.xlsx", index=False)  # not real Excel format

Correct — use to_excel 

df.to_excel("report.xlsx",
            index=False,
            sheet_name="Sales")  # proper Excel file
print("Excel file saved")        # confirm

Match the function to your audience: to_csv() for data pipelines, to_excel() for colleagues who live in Excel.

Forgetting to Install openpyxl Before Writing Excel

to_excel() raises a ModuleNotFoundError with no helpful message about what to install.

Pitfall

Pandas does not ship with the openpyxl engine by default, so to_excel() fails the first time you try it in a fresh environment. The fix is a single pip install before your script runs.

Wrong — missing dependency 

# openpyxl not installed
df.to_excel("report.xlsx")  # ModuleNotFoundError

Correct — install first 

pip install openpyxl          # one-time install
df.to_excel("report.xlsx", index=False)  # works now
print("Excel file saved")

Add openpyxl to your project's requirements.txt so it gets installed automatically when anyone sets up the environment.

Garbled Special Characters in CSV Opened in Excel

Accented names and currency symbols appear as question marks or boxes in Excel.

Pitfall

Excel on Windows assumes a CSV is encoded in the system's local code page, not UTF-8. If you write plain utf-8, accented characters like ñ or € get mangled when the file is opened. Use utf-8-sig to add a byte order mark (BOM) that tells Excel to read UTF-8 correctly.

Wrong — default encoding 

df.to_csv("intl.csv", index=False)  # Excel shows Ã© instead of é

Correct — use utf-8-sig 

df.to_csv("intl.csv",
          index=False,
          encoding="utf-8-sig")  # BOM tells Excel to use UTF-8
print("Saved with BOM encoding")

If your data only contains plain ASCII (A-Z, 0-9), the default encoding is fine. Use utf-8-sig whenever the data might contain names, currencies, or characters from non-English languages.
