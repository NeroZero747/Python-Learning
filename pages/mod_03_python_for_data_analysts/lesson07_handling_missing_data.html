Lesson Objective

The goal and expected outcome of this lesson

Detect missing values

The isnull function marks every missing cell so you can count how many gaps exist in each column.

Drop rows or columns

The dropna method removes any row or column that contains at least one null value from the DataFrame.

Fill missing values

The fillna method replaces null cells with a constant value, a column average, or an adjacent row value.

Pick the right strategy

Numeric columns usually benefit from mean imputation while categorical columns are better filled with the most common value.

This lesson prevents silent calculation errors that occur when missing data slips through into aggregations and summaries.

Overview

A high-level summary of the topic

Py

Missing data is a gap in your table where no value was recorded — and pandas gives you tools to find, remove, or fill those gaps.

Think of missing data like blank answers on a survey form — you can discard the form, skip the blank, or estimate a reasonable answer.

isnull()

The blank-field scanner — marks every empty cell

You call df.isna() to get a True/False mask of every cell in the DataFrame, then chain .sum() to count the gaps per column. This is the first step in any data-cleaning workflow because it tells you exactly which columns need attention before you run any calculations.

dropna()

The discard pile — removes forms with blanks

You call df.dropna() to remove any row (or column) that contains at least one NaN, or pass subset=["col"] to delete only rows where a specific column is blank. This is the fastest cleanup option when very few rows are affected, but it permanently removes data so you should always check how many rows you are about to lose.

fillna()

The best-guess pen — writes into every blank

You call df.fillna(value) to replace every NaN with a constant, the column mean, or even the previous row's value via forward-fill. This lets you keep all your rows instead of deleting them, which is essential when a large percentage of data is missing and dropping would shrink your dataset too much.

Imputation strategy

The estimation rule — how to pick the fill value

Your choice between dropping and filling depends on how much data is missing and what type the column holds — numeric columns usually benefit from the mean or median, while categorical columns are better filled with the most frequent value (mode). Choosing the wrong strategy can bias your results, so you should always check the missing percentage with isna().mean() before deciding.

One missing value in a numeric column turns the entire column's sum to NaN — always check for nulls before aggregating.

Key Takeaways

The most important ideas to remember

NaN Hides Inside Numeric Columns

A single NaN in a numeric column silently turns the result of sum() or mean() into NaN — the entire aggregate disappears without an error message. Running isna().sum() immediately after loading any file is the fastest way to spot these invisible gaps before they corrupt your results.

isna() Silent failure Detection

dropna() Removes Whole Rows by Default

Without a subset= parameter, bare dropna() deletes every row that has even one NaN in any column — a single blank "notes" field removes the entire record, including its valid data. Always pass a list of critical columns so you only drop rows where the blanks actually matter.

dropna() subset= Row loss

Forward-Fill Is Built for Ordered Data

For time-series data such as stock prices or daily sensor readings, ffill() copies the last known value into the gap — the most reasonable estimate when data arrives in chronological order. Using the column mean instead would inject an artificial "average day" into the middle of a trend, distorting any time-based analysis.

ffill() Time series Ordered data

The Right Strategy Depends on Why Data Is Missing

Filling a salary column with zero makes every employee look unpaid and pulls your averages down; filling it with the column mean hides the fact that data is incomplete. Checking the percentage of missing values first — under 5% favors dropping, above 5% favors filling — prevents both data loss and data fabrication.

Strategy % missing Bias risk

Key Concepts

isna(), dropna(), fillna(), and missing-data strategy.

Detecting Missing Data

Find NaN values

detection

Pandas represents missing values as NaN (Not a Number). Use isna() to create a boolean mask that is True wherever a value is missing.

Detection Methods

df.isna() True/False mask for every cell
df.isna().sum() Count of missing per column
df.notna() Opposite — True where not missing
Python 
print(df.isna().sum())        # missing count per column
print(df["Age"].isna().sum()) # missing in one column
print(df.isna().any())        # True if column has any NaN

Never use == None or == NaN. They do not detect Pandas missing values. Always use isna().

Removing Missing Data

Drop rows or columns with NaN

method

dropna() removes rows (or columns) that contain missing values. By default it drops any row with at least one NaN.

dropna() Parameters

axis=0 Drop rows (default)
axis=1 Drop columns
how="all" Drop only if every value is NaN
subset=["col"] Check only specific columns
Python 
clean = df.dropna()                     # drop any row with NaN
clean2 = df.dropna(subset=["Email"])    # drop only if Email is NaN
print(clean.shape)                       # fewer rows

dropna() returns a new DataFrame. The original is unchanged unless you add inplace=True or reassign: df = df.dropna().

Filling Missing Data

Replace NaN with a value

method

fillna() replaces every NaN with a value you choose. Pass a number, a string, or even a column's mean().

fillna() Options

df.fillna(0) Replace NaN with zero
df.fillna("Unknown") Replace NaN with text
df.fillna(df.mean()) Replace with column averages
df.fillna(method="ffill") Forward-fill from previous row
Python 
df["Age"] = df["Age"].fillna(df["Age"].mean())  # fill with average
df["City"] = df["City"].fillna("Unknown")       # fill with text
print(df.isna().sum())                            # confirm zero NaN

Save the result. df.fillna(0) alone does nothing. Write df["col"] = df["col"].fillna(0) to keep the change.

Choosing a Strategy

Drop or fill — when to use each

decision

Choosing a strategy depends on why the data is missing and how much of it is gone. Dropping works when very few rows are affected. Filling works when you have a sensible replacement value.

Drop vs Fill

dropna() fillna()
Best when Few rows missing Many rows missing
Risk Lose too much data Introduce fake values
Speed Very fast Very fast
Common fill — Mean, median, "Unknown"
Python 
pct = df.isna().mean() * 100       # % missing per column
print(pct)                          # decide based on percentage
# < 5% missing → dropna() is safe
# > 5% missing → fillna() is safer

Check the percentage first. If a column is 40% empty, dropping rows throws away nearly half your data. Use fillna() instead.

03

Fill Missing Values

Intermediate Scores fillna() mean()

What This Does

This script replaces missing values with the column's average using fillna(). This is useful when dropping rows would lose too much data.

fill_missing.py 
import pandas as pd  # load pandas

df = pd.read_csv("scores.csv")  # load test scores

avg_score = df["score"].mean()         # calculate the average
df["score"] = df["score"].fillna(avg_score)  # replace NaN with average

print(df["score"].isna().sum())  # confirm: should be 0
Terminal $ python fill_missing.py
0

Filling with the mean works well for normally distributed numbers — for skewed data, consider using the median instead.

04

Forward-Fill a Time Series

Intermediate Prices ffill()

What This Does

This script fills gaps in a daily price column by carrying the last known price forward — the standard approach for ordered time-series data.

forward_fill.py 
import pandas as pd  # load pandas

df = pd.read_csv("prices.csv")   # daily stock prices
print(df["close"].isna().sum())   # 4 missing trading days

df["close"] = df["close"].ffill()  # carry last known price forward
print(df["close"].isna().sum())    # 0 — all gaps filled
Terminal $ python forward_fill.py
4
0

ffill() only works when your data is sorted chronologically. If the first row is NaN, ffill() leaves it as NaN because there is no previous value to copy.

05

Choose Strategy by Percentage

Intermediate Employees isna().mean() Decision

What This Does

This script calculates the percentage of missing values per column and applies a different strategy based on that percentage — drop when < 5%, fill when ≥ 5%.

strategy_check.py 
import pandas as pd  # load pandas

df = pd.read_csv("employees.csv")       # load employee data
pct = df.isna().mean() * 100            # % missing per column
print(pct.round(1))                      # display percentages

# Drop columns with very few gaps
df = df.dropna(subset=["employee_id"])   # < 1% missing — safe to drop

# Fill columns with many gaps
median_salary = df["salary"].median()    # robust central value
df["salary"] = df["salary"].fillna(median_salary)  # 18% missing — fill

print(df.isna().sum())  # confirm cleanup

The 5% threshold is a practical guideline, not a hard rule. Always consider the column's business importance — a primary key column should never be filled, even if 20% is missing.

If you already handle NULL values in SQL or work with empty cells in Excel, pandas uses NaN to represent missing data — and the same three approaches apply: detect it, drop it, or fill it.

Intermediate

Your Task

You have a customer dataset with missing values in several columns. Calculate the percentage of missing values per column with isna().mean() * 100, then apply dropna() to columns under 5% and fillna() to the rest.

pick_strategy.py 
import pandas as pd  # load pandas

df = pd.read_csv("customers.csv")        # customer data
pct = df.isna().mean() * 100             # % missing per column
print(pct.round(1))                       # review percentages

# Drop rows where 'email' is missing (only 2% gaps)
df = df.dropna(subset=["email"])

Common Mistakes

Pitfalls beginners hit when working with missing data

Comparing to None With the == Operator

The comparison always returns False because NaN is not equal to anything, including itself.

Pitfall

NaN == NaN returns False in Python. Using df["Age"] == None finds nothing. Use .isnull() instead.

Wrong — == None 

missing = df[df["Age"] == None]   # finds nothing
print(len(missing))                # 0 — even if NaNs exist

Correct — use .isnull() 

missing = df[df["Age"].isnull()]  # detects NaN correctly
print(len(missing))                # actual count of blanks

NaN is a special value that is not equal to anything — not even itself. Always use .isnull() or .isna() to find missing values.

Using dropna() Without Specifying a Column

Rows are dropped even if only one irrelevant column has a missing value.

Pitfall

Bare df.dropna() drops every row that has a blank in any column. If only one column matters, pass subset=["column"] to keep the rest.

Wrong — drop all blanks 

clean = df.dropna()       # drops rows with ANY blank
print(clean.shape)         # far fewer rows than expected

Correct — target one column 

clean = df.dropna(subset=["Age"])  # only drop if Age is blank
print(clean.shape)                  # keeps most of the data

Think of dropna() without a subset as throwing away an entire form because one optional field is blank. Target the column that actually matters.

Calling fillna() Without Saving the Result

The original DataFrame still contains NaN because fillna() returns a new copy.

Pitfall

fillna() does not change the DataFrame in place by default. Assign the result back to the column or pass inplace=True.

Wrong — result discarded 

df["Age"].fillna(0)      # returns a copy — original unchanged
print(df["Age"].isnull().sum())   # still has NaN

Correct — assign back 

df["Age"] = df["Age"].fillna(0)   # save the filled version
print(df["Age"].isnull().sum())    # 0 — no more blanks

Most Pandas methods return a new object. If you do not assign the result, the change is lost — like editing a document without pressing Save.

Filling Every Column With Zero

Zero is not always a neutral value — it pulls averages down and turns missing salaries into "unpaid."

Pitfall

Writing df.fillna(0) on the whole DataFrame converts every blank into zero, regardless of what the column represents. A missing salary becomes $0, dragging the team's average salary down and inflating the count of low earners.

Wrong — blanket zero fill 

df = df.fillna(0)          # salary blanks → 0
print(df["salary"].mean())  # average drops sharply

Correct — column-specific fill 

df["salary"] = df["salary"].fillna(df["salary"].median())
print(df["salary"].mean())  # realistic average

Choose a fill value that makes sense for each column: median for skewed numbers, mode for categories, and zero only when zero genuinely means "none" (e.g. quantity sold).

Aggregating Before Checking for NaN

Pandas silently skips NaN in sum() and mean() — your totals look fine but are actually incomplete.

Pitfall

By default, df.sum() and df.mean() skip NaN rows without any warning. The result looks valid but is calculated from fewer rows than you expect, silently under-counting totals or averaging over a smaller sample.

Wrong — aggregate blindly 

total = df["revenue"].sum()   # skips NaN silently
print(f"Total: ${total:,.0f}")  # looks right, but low

Correct — check first 

print(df["revenue"].isna().sum())  # 12 missing
df["revenue"] = df["revenue"].fillna(0)
total = df["revenue"].sum()  # now includes all rows

Always run isna().sum() on a column before using sum() or mean(). If any values are missing, handle them first so your totals reflect the full dataset.
