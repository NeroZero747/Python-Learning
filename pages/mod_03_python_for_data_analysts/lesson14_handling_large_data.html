Lesson Objective

The goal and expected outcome of this lesson

Why DataFrames use so much RAM

Pandas defaults to 64-bit types for every numeric column, consuming far more memory than the actual values require.

Inspect memory column by column

The memory_usage method shows how many bytes each column consumes so you can target the largest ones first.

Downcast numeric columns

Converting a float64 column to float32 or an int64 to int16 can cut that column's memory use in half.

Use categorical columns

Switching string columns to the categorical type stores repeated values once instead of once per row, slashing memory.

This lesson teaches you to cut RAM usage so your script can handle datasets that would otherwise crash Python.

Key Concepts

memory_usage(), data types, downcasting, and the category dtype.

memory_usage()

Check how much RAM a DataFrame uses

method

memory_usage() shows how many bytes each column uses. Add deep=True to count the actual memory of string columns.

Key Parameters

No arguments Approximate sizes (fast)
deep=True Exact sizes — counts string content
.sum() Total bytes across all columns
/ 1e6 Convert bytes to megabytes
Python 
print(df.memory_usage(deep=True))              # bytes per column
total_mb = df.memory_usage(deep=True).sum() / 1e6
print(f"Total: {total_mb:.1f} MB")             # human-readable

Always pass deep=True for DataFrames with text columns. Without it, pandas reports the pointer size, not the actual string content.

Data Types

How pandas stores each column

concept

Every column has a data type (dtype) that controls memory usage. An int64 column uses 8 bytes per value. An int8 uses only 1 byte — eight times less.

Common Pandas dtypes

Op Meaning Example
int64 Default integer — 8 bytes df["id"].dtype
int32 Smaller integer — 4 bytes df["age"].astype("int32")
float64 Default decimal — 8 bytes df["price"].dtype
category Repeated strings — very compact df["status"].astype("category")
Python 
print(df.dtypes)                               # current types
print(df["age"].dtype)                         # int64
df["age"] = df["age"].astype("int8")           # downcast to 1 byte

Check df.dtypes right after loading. Pandas often assigns int64 to columns that only hold small numbers like ages or quantities.

Downcasting

Switch to a smaller numeric type

technique

Downcasting means converting a column from a large type like int64 to a smaller one like int16 or int8. Use pd.to_numeric() with downcast to let pandas pick the smallest safe type.

Before vs After Downcast

int64 (before) int8 (after)
Bytes per value 8 1
Max value 9.2 quintillion 127
1M rows 8 MB 1 MB
When safe Always Values fit in range
Python 
df["qty"] = pd.to_numeric(df["qty"], downcast="integer")
df["age"] = df["age"].astype("int8")           # max 127
df["year"] = df["year"].astype("int16")        # max 32,767

Check the column max value before downcasting. Run df["col"].max() — if the maximum is 150, int8 (max 127) is too small.

Category dtype

Efficient storage for repeated text

dtype

The category dtype stores repeated string values as integer codes with a lookup table. A column with 1 million rows but only 5 unique values drops from hundreds of megabytes to under 1 MB.

object vs category

object (string) category
Storage Full text per row Integer code + lookup
Memory ✗ High for repeats ✓ Very low
Best for Unique text (names) Repeated labels (region)
Convert Default type astype("category")
Python 
print(df["region"].memory_usage(deep=True))    # before: large
df["region"] = df["region"].astype("category") # convert
print(df["region"].memory_usage(deep=True))    # after: much smaller

Use category for columns with fewer than 50 unique values. Columns like "status", "region", or "department" are ideal candidates.

chunksize Parameter

Load data in small batches

parameter

The chunksize parameter tells pd.read_csv() to load the file in batches instead of all at once. Each batch is a DataFrame with that many rows.

How chunksize Works

chunksize=10000 Load 10,000 rows at a time
Returns A TextFileReader iterator, not a DataFrame
Each chunk A regular DataFrame you can process
Memory Only one chunk lives in RAM at a time
Python 
reader = pd.read_csv("big.csv", chunksize=10000)
for chunk in reader:                           # one chunk at a time
    print(chunk.shape)                         # (10000, 8)

Start with chunksize=10000 and adjust. Smaller chunks use less memory but run more loops. Larger chunks are faster but need more RAM.

Chunk Iterator

Loop through chunks one by one

concept

A chunk iterator is the object returned by read_csv(chunksize=N). It yields one DataFrame per iteration. You cannot go backwards — each chunk is read once and then discarded.

Full Load vs Chunk Iterator

Full load Chunked
Returns One DataFrame An iterator of DataFrames
Memory ✗ Entire file in RAM ✓ One chunk in RAM
Access Random (any row) Sequential (forward only)
Best for Small – medium files Files larger than RAM
Python 
reader = pd.read_csv("big.csv", chunksize=50000)
for i, chunk in enumerate(reader):             # numbered chunks
    print(f"Chunk {i}: {len(chunk)} rows")     # progress

You cannot rewind a chunk iterator. Once you loop through it, the data is gone. If you need to iterate twice, open a new reader.

Aggregating Chunks

Combine results from all chunks

technique

Aggregating chunks means computing a result across the entire file by accumulating values from each chunk. Collect partial results in a running total, then read the final value after the loop.

Aggregation Strategies

Sum Keep a running total across chunks
Count Increment a counter per chunk
Collect Append filtered rows to a list
GroupBy Concat partial results, groupby once at end
Python 
total = 0                                      # running total
reader = pd.read_csv("big.csv", chunksize=10000)
for chunk in reader:                           # loop all chunks
    total += chunk["sales"].sum()              # accumulate
print(f"Grand total: {total}")                 # final result

For grouped aggregations, collect partial results and concat at the end. Do not call groupby() inside the loop — it creates incomplete groups.

When to Use Chunks

Decide whether chunking is necessary

guideline

Use chunk processing when a file is too large to fit in memory. If read_csv() raises a MemoryError or your system freezes, switch to chunked reading. For files under 500 MB, full loading is usually fine.

Full Load vs Chunked

Full load Chunked
File size Under 500 MB Over 500 MB
Speed ✓ Faster (single read) ✗ Slower (loop)
Complexity ✓ Simple code ✗ More complex
Memory safety ✗ May crash ✓ Always safe
Python 
import os
size_mb = os.path.getsize("data.csv") / 1e6    # file size in MB
if size_mb > 500:                              # large file
    reader = pd.read_csv("data.csv", chunksize=50000)
else:
    df = pd.read_csv("data.csv")               # load normally

Check your available RAM before deciding. A 2 GB file fits in memory on a 16 GB machine but crashes on a 4 GB machine.

Vectorised Operations

Apply operations to entire columns at once

technique

A vectorised operation applies a calculation to every value in a column in one step, without a loop. Pandas does the looping internally in compiled C code, which is far faster than a Python for loop.

Loop vs Vectorised

for loop Vectorised
Code for i in range(len(df)): df["x"] = df["a"] * 2
Speed ✗ Seconds to minutes ✓ Milliseconds
Lines 3+ lines 1 line
When Complex custom logic Standard math and comparisons
Python 
# Slow — Python loop
# for i in range(len(df)):
#     df.loc[i,"total"] = df.loc[i,"qty"] * df.loc[i,"price"]
# Fast — vectorised
df["total"] = df["qty"] * df["price"]          # all rows at once

If you write a for loop over DataFrame rows, stop and look for a vectorised alternative. Pandas has built-in methods for almost every common operation.

Early Filtering

Remove rows before processing

technique

Early filtering means dropping unwanted rows as soon as possible after loading. Fewer rows means every later operation — groupby, merge, export — runs faster and uses less memory.

Filtering Strategies

usecols=[...] Skip unwanted columns at load time
df[condition] Drop rows by value after loading
df.dropna() Remove rows with missing values
df[date >= ...] Limit to a date range
Python 
df = pd.read_csv("orders.csv")                 # load data
df = df[df["status"] == "completed"]            # keep completed
df = df[df["date"] >= "2024-01-01"]             # recent only
print(f"Kept {len(df)} rows")                   # check reduction

Filter before groupby or merge. A groupby on 1 million rows is much faster than on 10 million rows. Remove what you do not need first.

Avoiding Copies

Reduce memory by not duplicating data

technique

Avoiding copies means modifying data in place instead of creating new DataFrames. Every new assignment duplicates memory. On large datasets this can double your RAM usage.

Copy vs In-Place

Creates copy In place
Memory ✗ 2× original ✓ Same as original
Original Unchanged Modified
Safety ✓ Can undo ✗ Cannot undo
When Need both versions Only need final result
Python 
# Creates a copy — uses extra memory
# df_clean = df[df["status"] == "active"].copy()
# Modifies in place — saves memory
df.drop(columns=["temp_col"], inplace=True)    # remove column
df.reset_index(drop=True, inplace=True)        # reset index

Use inplace=True only when you are certain you do not need the original data. It saves memory but the change cannot be undone.

Method Chaining

Combine operations in one statement

pattern

Method chaining connects multiple pandas operations with dots in a single statement. Each method returns a DataFrame, so you can call the next method on the result without intermediate variables.

Separate Steps vs Chain

Separate variables Chained
Variables 3+ temporary names 1 final name
Memory ✗ Stores each step ✓ Only final result
Readability Easy to debug Compact and clean
Best for Learning and debugging Production scripts
Python 
result = (
    df.dropna(subset=["sales"])                 # remove nulls
    .query("sales > 0")                         # filter positives
    .groupby("region")["sales"].sum()           # aggregate
    .reset_index()                               # flatten
)

Wrap the chain in parentheses. This lets you break it across multiple lines. Without parentheses, Python requires backslashes at the end of each line.

Check Memory Usage

Beginner Sales memory_usage() deep=True

What This Does

This script loads a CSV and prints how many megabytes each column uses. You can spot oversized columns before deciding where to optimize.

check_memory.py 
import pandas as pd  # load pandas

df = pd.read_csv("sales.csv")  # load sales data

print(mem_mb.round(2))

02

Downcast Numbers

Beginner Sensors pd.to_numeric() downcast

What This Does

This script converts a 64-bit integer column to the smallest type that can hold its values, cutting memory in half or more.

downcast_int.py

02

Filter Each Chunk

Beginner Transactions filter append results

What This Does

This script filters each chunk for rows that match a condition and collects only the matching rows. The full file never sits in memory at once.

filter_chunks.py 
import pandas as pd  # load pandas

results = []  # collect matching rows

for chunk in pd.read_csv("txns.csv", chunksize=50_000):
    high = chunk[chunk["amount"] > 1000]  # filter this chunk
    results.append(high)                    # keep matches

df = pd.concat(results, ignore_index=True)  # combine
print(f"Found {len(df)} high-value transactions")
Terminal $ python filter_chunks.py
Found 2,347 high-value transactions

Filtering inside the loop keeps only the rows you need — the rest are discarded before the next chunk arrives.

Replace a For-Loop

What This Does

Common Mistakes

Pitfalls beginners hit when working with memory optimization

Loading Data With Default int64 and float64 Types

Each column uses 8 bytes per value when 1–4 bytes would suffice.

Pitfall

Pandas defaults to int64 and float64. Downcasting to int32 or float32 can cut memory use in half with no precision loss for most analytics.

Wrong — default 64-bit 

df = pd.read_csv("big_data.csv")
print(df.memory_usage(deep=True).sum())  # very large

Correct — downcast types 

df = pd.read_csv("big_data.csv")               # load first
df["age"] = pd.to_numeric(
    df["age"], downcast="integer")               # int64 → int8
print(df.memory_usage(deep=True).sum())          # much smaller

Downcasting is like choosing a carry-on bag instead of a suitcase when you only have a few items. Same contents, less space.

Leaving Repeated Text as object Instead of category

Object columns store every string separately, consuming far more memory than necessary.

Pitfall

A column with few unique values (like "Yes"/"No" or country names) wastes memory as object. Converting to category stores each unique value once.

Wrong — object type 

print(df["country"].dtype)        # object — full string copy
print(df["country"].memory_usage(deep=True))  # 8 MB

Correct — use category 

df["country"] = df["country"].astype("category")  # convert
print(df["country"].dtype)                          # category
print(df["country"].memory_usage(deep=True))         # 0.5 MB

If a column has fewer than ~1,000 unique values out of millions of rows, category is almost always a win.

Optimizing Before Measuring Current Memory Usage

You waste time converting columns that were already small.

Pitfall

Always run df.memory_usage(deep=True) first to find the biggest offenders. Optimize the largest columns first for the biggest gains.

Wrong — guess and convert 

df["id"] = df["id"].astype("int8")         # already tiny
df["flag"] = df["flag"].astype("category") # already tiny

Correct — measure first 

mem = df.memory_usage(deep=True)    # measure each column
print(mem.sort_values(ascending=False).head())  # biggest first
# then optimize only the top offenders

Measuring before optimizing is like checking your bank statement before cutting expenses. Fix the biggest leak first.
