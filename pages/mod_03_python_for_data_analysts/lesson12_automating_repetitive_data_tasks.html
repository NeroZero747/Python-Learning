Key Takeaways

The most important ideas to remember

Write Idempotent Scripts

A well-designed automation script produces the same output whether it runs once or ten times; always overwrite output files rather than appending rows, to avoid duplicating data over time.

Idempotent Overwrite No duplicates

Alert on Failure, Not Success

Logging a message only when the script fails means you do not need to check a log every day — silence means everything ran correctly, and a notification means action is required.

Failure alerts Silent success Notifications

Version-Control Automation Scripts

Storing your automation scripts in Git means you can roll back to last week's working version in thirty seconds if a code change breaks a scheduled run before the next run is due.

Git Roll back 30 seconds

Key Concepts

glob(), file loops, pd.concat(), and reusable functions.

glob()

Find files by name pattern

function

glob() searches a folder for files matching a pattern. Use * to match any characters. It returns a list of file paths you can loop through.

Common Glob Patterns

*.csv All CSV files in the folder
data_*.xlsx Excel files starting with data_
**/*.csv CSV files in all subfolders
sales_202?.csv sales_2020 through sales_2029
Python 
files = glob.glob("data/*.csv")              # all CSVs in data/
print(len(files))                             # how many found
print(files[0])                               # first file path

Add recursive=True to search subfolders. Without it, the ** pattern does not descend into nested directories.

File Loops

Process every file in a list

pattern

A file loop iterates over a list of file paths and runs the same code on each one. Combine glob() with a for loop to process an entire folder automatically.

Loop Pattern Steps

Step 1 Find files with glob()
Step 2 Create an empty list for results
Step 3 Loop and read each file
Step 4 Append each DataFrame to the list
Python 
files = glob.glob("reports/*.csv")            # find all CSVs
frames = []                                   # empty collector
for f in files:                               # loop each file
    frames.append(pd.read_csv(f))             # load and collect

Print the file name inside the loop while developing. A quick print(f) shows which file caused an error if something fails.

pd.concat()

Stack multiple DataFrames into one

function

pd.concat() joins a list of DataFrames into one. By default it stacks them vertically. All frames must share the same column names.

concat() vs merge()

concat() merge()
Direction Vertical — stack rows Horizontal — add columns
Input List of DataFrames Two DataFrames
Match on Same column names A shared key column
Use for Combining monthly files Joining related tables
Python 
all_data = pd.concat(frames)                  # stack all frames
all_data = pd.concat(frames, ignore_index=True)  # reset index
print(all_data.shape)                         # total rows and cols

Add ignore_index=True to reset row numbers. Without it, row numbers from each file overlap (0, 1, 2, 0, 1, 2…).

Reusable Functions

Wrap logic in a function for reuse

pattern

A reusable function wraps your automation logic so you can call it with different inputs. Write the steps once inside def, then call the function with any folder or pattern.

Function Benefits

Write once Define the logic a single time
Call many times process_folder("jan/"), process_folder("feb/")
Easy to test Run with a single test file first
Easy to read A function name describes the intent
Python 
def process_folder(pattern):                  # define once
    files = glob.glob(pattern)                # find matching files
    frames = [pd.read_csv(f) for f in files]  # load each one
    return pd.concat(frames, ignore_index=True)  # combine

Name functions after what they do, not how they do it. Use process_monthly_sales() instead of loop_and_concat().

Code Examples

Hands-on code snippets to explore the concepts

01

Loop Over Files

Beginner Reports glob() for loop

What This Does

This script finds all CSV files in a folder and reads each one in a loop. You process any number of files without writing a separate line for each.

loop_files.py 
import pandas as pd
from pathlib import Path

folder = Path("reports/")  # folder containing CSVs

for file in folder.glob("*.csv"):              # loop over CSV files
    df = pd.read_csv(file)                      # read each file
    print(f"{file.name}: {len(df)} rows")      # show file name and size
Terminal $ python loop_files.py
jan_report.csv: 120 rows
feb_report.csv: 145 rows

glob("*.csv") returns every CSV in the folder — add new files and the script picks them up automatically.

02

Combine and Summarize

Beginner Sales pd.concat() append results

What This Does

This script reads multiple files, stacks them into one DataFrame with pd.concat(), then calculates a grand total.

combine_files.py 
import pandas as pd
from pathlib import Path

frames = []  # empty list to collect DataFrames
for file in Path("sales/").glob("*.csv"):  # loop over files
    frames.append(pd.read_csv(file))         # add each to the list

all_sales = pd.concat(frames, ignore_index=True)  # stack together
print("Total revenue:", all_sales["revenue"].sum())
Terminal $ python combine_files.py
Total revenue: 87450.0

Collect DataFrames in a list and concat once at the end — it is much faster than appending inside the loop.

Log the Results

Beginner ETL logging automation

What This Does

This script processes files and writes a log message for each one. When the script runs unattended, logs tell you what happened.

log_results.py 
import pandas as pd
import logging

logging.basicConfig(filename="etl.log", level=logging.INFO)

Logging writes to a file so you can check results later — print() output disappears as soon as the terminal closes.

Common Mistakes

Pitfalls beginners hit when working with automating data tasks

Hardcoding File Paths Instead of Using Patterns

The script breaks the moment a new file is added or a name changes.

Pitfall

Listing every filename by hand is fragile. Use glob.glob() to match file patterns dynamically.

Wrong — manual list 

files = ["jan.csv", "feb.csv", "mar.csv"]   # breaks in April
for f in files:
    df = pd.read_csv(f)

Correct — use glob 

from glob import glob                    # pattern matcher
files = glob("data/*.csv")               # finds all CSVs
for filepath in files:                    # loop dynamically
    df = pd.read_csv(filepath)            # handles any new file

glob() is like telling your assistant "process every CSV in this folder" instead of handing them a handwritten list that goes stale.

Letting One Bad File Crash the Entire Loop

A single corrupt file stops the script before it processes the remaining files.

Pitfall

Without a try/except block, one bad file raises an exception and halts everything. Wrap the loop body so the script skips failures and continues.

Wrong — crash on error 

for filepath in files:
    df = pd.read_csv(filepath)   # crashes on corrupt file
    process(df)

Correct — skip and log 

for filepath in files:                     # loop all files
    try:
        df = pd.read_csv(filepath)         # attempt read
        process(df)                         # do the work
    except Exception as err:
        print(f"Skipped {filepath}: {err}") # log and continue

An automation script should be like a postman — if one letterbox is jammed, skip it and deliver the rest instead of going home.

Running a Script With No Output or Log Messages

When something goes wrong, there is no trail to follow — nothing tells you what ran or failed.

Pitfall

Silent scripts are hard to debug. Add print() statements (or logging) at key steps so you can trace what happened after the script finishes.

Wrong — silent script 

for filepath in files:
    df = pd.read_csv(filepath)
    df.to_excel(filepath.replace(".csv", ".xlsx"))

Correct — log progress 

for filepath in files:
    print(f"Processing {filepath}...")       # log start
    df = pd.read_csv(filepath)              # load
    out = filepath.replace(".csv", ".xlsx")  # output name
    df.to_excel(out, index=False)            # save
print(f"Done — {len(files)} files processed") # log total

Print statements are breadcrumbs through the forest. When something goes wrong at 2 AM, those breadcrumbs show you exactly where the script stopped.

Without automation
