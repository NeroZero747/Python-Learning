Key Takeaways

The most important ideas to remember

DataFrames Write to SQL

Calling df.to_sql('table', engine) pushes your entire DataFrame into a database table in one step — no row-by-row INSERT loop required.

to_sql() SQLAlchemy Bulk

Upsert Prevents Duplicates

An upsert inserts new rows and updates existing ones in a single operation, so re-running your pipeline does not create duplicate records.

INSERT OR UPDATE Idempotent Safe

Transactions Protect Integrity

Wrapping your writes in a transaction means either all rows commit or none do — a half-loaded table is worse than no load at all.

BEGIN/COMMIT Rollback Atomic

Code Examples

Hands-on code snippets to explore the concepts

Example 1 — Converting API Data into a DataFrame

Beginner
example_1_converting_api_data_into_a_dataframe.py 
import requests
import pandas as pd

url = "https://api.example.com/data"

response = requests.get(url)

data = response.json()

records = data["results"]

df = pd.DataFrame(records)

print(df.head())
02

Example 2 — Writing Data to a SQL Database

Beginner

What This Does

Pandas provides the to_sql() function.

example_2_writing_data_to_a_sql_database.py 
from sqlalchemy import create_engine

engine = create_engine("sqlite:///example.db")

df.to_sql("sales_data", engine, if_exists="replace", index=False)

This creates a table in the database.

03

Example 3 — Appending Data to Existing Tables

Beginner

What This Does

This adds new records without replacing existing data.

example_3_appending_data_to_existing_tables.py 
df.to_sql("sales_data", engine, if_exists="append", index=False)
04

Example 4 — Querying Stored Data

Beginner

What This Does

Once data is stored, it can be queried with SQL.

example_4_querying_stored_data.sql 
SELECT city, SUM(sales)
FROM sales_data
