Overview

A high-level summary of the topic

Py

An if statement is Python's way of asking "Is this true?" — and deciding exactly what to do next based on the answer.

Think of an if statement like a set of traffic lights at a busy junction — it reads the current conditions and gives exactly one instruction: Go, Slow, or Stop. An if / elif / else block works the same way: Python checks each condition in order and follows the first path that matches.

The if condition

The sensor — reads whether a condition is True

The opening question Python evaluates before deciding which branch to run.

Indented code block

The green light — the action that runs when True

Every statement inside this block runs when the condition above it is True.

The elif clause

The second checkpoint — a different condition

A follow-up question when the first check comes back no.

The else clause

The waiting area — the catch-all fallback

The default pathway when every check above was false.

This same pattern maps directly to SQL's CASE WHEN statement — every if / elif / else block you write in Python translates one-for-one into queries you may already be running.

Key Concepts

The if, elif, and else keywords — what each one does and when to use it.

if Statement

Runs a block only when a condition is True

if

An if statement is a conditional gate that runs its indented block only when the condition is True. Write the keyword if, then a condition, then a colon — Python evaluates the condition and skips the block entirely when it is False.

if Statement Syntax Rules

Start with keyword if if age >= 18: ✓   age >= 18: ✗
Use == to compare age == 18 ✓   age = 18 ✗
End line with colon : if age >= 18: ✓   if age >= 18 ✗
Python 
age = 20                        # the customer's age

if age >= 18:                   # condition — is this True?
    print("Access granted")     # runs only when True
# nothing happens if age is below 18

Every if line must end with a colon. Forgetting it causes a SyntaxError immediately — Python requires the colon to know an indented block is about to start.

elif Statement

Chains additional conditions after if

elif

elif — short for "else if" — chains an additional condition that Python only evaluates when every condition above it was False. You can stack as many elif branches as you need; Python runs the first one that is True and skips the rest.

Comparison Operators for Conditions

Op Meaning Example
== Equal to score == 80 → True
!= Not equal to score != 0 → True
>= At least score >= 60 → True
> Greater than score > 80 → False
<= At most age <= 12 → True
< Less than items < 5 → True
Python 
score = 74                      # quiz score as a percentage

if score >= 80:                 # check highest threshold first
    print("Distinction")
elif score >= 60:               # only reached if line above was False
    print("Pass")              # ← this runs; 74 >= 60 is True
elif score >= 0:                # skipped — a True branch already ran
    print("Try again")

Order your elif thresholds from most restrictive to least. If you wrote elif score >= 0 first, it would match every score and the Distinction and Pass thresholds would never run.

else Statement

Catch-all fallback when no condition matched

else

The else clause is a catch-all fallback that runs only when every condition above it was False. It has no condition of its own — it simply fires when nothing else matched, ensuring your code always produces a defined outcome even for unexpected input values.

Without else vs With else

No else With else
Input example ticket = "Basic" ticket = "Basic"
No match → output ✗ Nothing prints ✓ Standing area
Data safety Possible silent gap Always handled
Python 
ticket_type = "Basic"           # the ticket category chosen by the customer

if ticket_type == "Premium":
    print("Access all areas — VIP lounge included")
elif ticket_type == "Standard":
    print("Main hall access")
else:                           # no condition — catches everything else
    print("Standing area only") # ← runs for Basic, typos, unexpected values

Always include an else clause in production rules. Real data contains typos, nulls, and unexpected values — without else, those records silently produce no output, which is harder to debug than an explicit fallback message.

Decision Flow

How Python works through an if / elif / else block step by step

The golden rule: Python checks each condition top to bottom and runs the first block that matches — the moment one branch executes, all remaining branches are skipped automatically.

How Python steps through the block

1
if ticket_type == "Premium" evaluated first — always
YES Run this block, then stop
NO Skip — check elif next
if was False — move to elif
2
elif ticket_type == "Standard" only reached if if was False
YES Run this block, then stop
NO Skip — fall to else
elif was also False — move to else
3
else no condition needed only reached when all above were False
ALWAYS No condition to check — always runs if reached

Example walkthrough — ticket_type = "Premium"

if ticket_type == "Premium" → True
✓ RUNS
elif ticket_type == "Standard" never evaluated
✗ SKIPPED
else never reached
✗ SKIPPED
ticket_check.py 
ticket_type = "Premium"

if ticket_type == "Premium":
    print("Access all areas — VIP lounge included")  # ← This runs — condition is True
elif ticket_type == "Standard":
    print("Main hall access")              # ← Skipped
else:
    print("Standing area only")           # ← Skipped
Output
Access all areas — VIP lounge included

Because ticket_type == "Premium" is True, Python runs that block immediately and skips the elif and else entirely — they are never evaluated.

Code Examples

Hands-on code snippets to explore the concepts

01

Age Check

Beginner if / else >=

What This Does

This script stores a customer's age as a variable and uses an if/else statement to print one of two messages — one when the condition is True, the other when it's False.

age_check.py 
age = 20                        # the customer's age

if age >= 18:                   # check if the condition is True
    print("Access granted")     # runs only when age is 18 or above
else:                           # runs when the condition is False
    print("Access denied — must be 18 or over")
Terminal $ python age_check.py
Access granted

Change age = 20 to age = 16 and Python will automatically run the else branch instead — no other line needs to change.

02

Delivery Band

Beginner if / elif / else >=

What This Does

This script uses if/elif/else to assign the correct delivery cost based on the basket total. Python checks each threshold from top to bottom and runs the first branch that is True.

delivery_band.py 
basket_total = 35              # the customer's basket total in pounds

if basket_total >= 50:        # check the highest threshold first
    delivery = "Free delivery"
elif basket_total >= 20:      # only reached if the line above was False
    delivery = "£2.99 standard delivery"  # ← this runs; 35 >= 20 is True
else:                          # basket below £20 — highest delivery charge
    delivery = "£4.99 delivery"

print(delivery)                # display the delivery cost
Terminal $ python delivery_band.py
£2.99 standard delivery

Try changing the basket total to 55 or 15 — Python will automatically pick a different branch each time without you needing to change anything else in the script.

03

Order Rating

Beginner if / elif / else + or

What This Does

This script checks two delivery metrics — minutes late and number of complaints — and uses or inside each condition so that either metric alone can trigger a refund action.

order_rating.py 
minutes_late = 25              # minutes after the estimated delivery time
complaints = 2                 # complaints from this restaurant in the last week

if minutes_late >= 30 or complaints >= 5:   # severe issue — either triggers full refund
    action = "Full refund issued"
elif minutes_late >= 10 or complaints >= 2: # minor issue — either triggers credit
    action = "10% credit added to your account"  # ← this runs
else:                                           # no issues — no action needed
    action = "Delivery on time — no action needed"

print(action)                  # display the outcome
Terminal $ python order_rating.py
10% credit added to your account

Using or means only one of the two checks needs to be True to trigger the action — raise minutes_late to 30 and watch the output jump to a full refund.

SQL / Excel Comparison

How this topic compares across Python, SQL, and Excel

If you already write CASE WHEN in SQL or use =IF() in Excel, you are already thinking in conditional logic — Python's if/elif/else block is the same idea, just written as plain readable code instead of a query clause or a nested formula.

Python
SQL
Excel
branch on one condition
Python if / else

Python runs the if block when the condition is true and the else block when it is false.

SQL CASE WHEN ... ELSE

SQL evaluates a WHEN clause and falls back to the ELSE value when no condition matches.

Excel =IF(test, yes, no)

Excel's IF checks a cell condition and returns one of two values depending on the result.

branch on multiple conditions
Python elif

You can add as many elif blocks as you need — Python checks each one in order until one is true.

SQL WHEN ... THEN (multiple)

Add more WHEN clauses inside CASE to handle additional conditions before the ELSE.

Excel =IFS(test1, val1, ...)

IFS tests each condition in order and returns the value paired with the first true condition.

default when nothing matches
Python else

The else block runs only when every condition above it was false — it is the safety net.

SQL ELSE

The ELSE clause at the end of CASE captures any row that did not match a WHEN condition.

Excel last IF argument

The third argument in =IF() is the fallback value returned when the test is false.

Same condition check, three tools

Python code

Python 
if ticket_type == "Premium":
    tier = "VIP"
elif ticket_type == "Standard":
    tier = "Main hall"
else:
    tier = "Standing"

SQL query

SQL 
CASE
  WHEN ticket_type = 'Premium'
    THEN 'VIP'
  WHEN ticket_type = 'Standard'
    THEN 'Main hall'
  ELSE 'Standing'
END AS tier

Excel formula

Excel 
=IF(A2="Premium","VIP",
  IF(A2="Standard","Main hall",
    "Standing"))

All three classify a ticket type into an access tier — the same result, just written in a different tool.

The if/elif/else pattern maps directly onto CASE WHEN in SQL and nested IF() in Excel — once you understand how Python checks conditions in order, you will recognize the same pattern immediately in your existing queries and spreadsheets.

Practice Exercises

Guided exercises to reinforce your learning

01

Check a Ticket

Beginner Cinema if / else

Your Task

A cinema ticket scanner checks whether a ticket type is "Standard" before letting someone into the screening. Set ticket_type to "Standard" and write an if/else block that prints "Welcome — enjoy the film!" when the ticket matches and "Ticket type not recognized" for anything else. Your output should show Welcome — enjoy the film!

ticket_check.py 
ticket_type = "Standard"          # store the scanned ticket type

if ticket_type == "Standard":     # check whether the ticket matches exactly
    print("Welcome — enjoy the film!")
else:                              # any other ticket type is turned away
    print("Ticket type not recognized")
Terminal $ python ticket_check.py
Welcome — enjoy the film!

Python compares strings character by character — "standard" and "Standard" are not equal because the capital letter differs. Always match the capitalization exactly when checking strings.

02

Grade a Score

Beginner Quiz if / elif / else

Your Task

An online quiz shows a grade label next to your final score. Set score to 74 and write an if/elif/else block that prints "Distinction" for 80 and above, "Pass" for 60 and above, and "Try again" for anything lower. Your output should show Pass.

grade_score.py 
score = 74                       # the learner's final quiz score

if score >= 80:                  # top band — distinction
    print("Distinction")
elif score >= 60:                # middle band — pass
    print("Pass")
else:                            # below pass mark
    print("Try again")
Terminal $ python grade_score.py
Pass

Python checks each condition in order and stops at the first one that is true. Always put the highest threshold first — if you put score >= 60 before score >= 80, every distinction would be labelled "Pass" instead.

03

Book a Seat

Beginner Travel if / elif / else + or

Your Task

A train booking app checks whether a seat is available and affordable before confirming a reservation. Set seats_left to 0 and price to 45, then write an if/elif/else block using or that prints "Cannot book" when there are no seats left or the price is above 100, "Booking confirmed" when seats are available and the price is 50 or below, and "Confirm with customer" otherwise. Your output should show Cannot book.

book_seat.py 
seats_left = 0                              # number of seats still available
price = 45                                  # ticket price in pounds

if seats_left == 0 or price > 100:          # no seats, or too expensive → block
    print("Cannot book")
elif seats_left > 0 and price <= 50:        # seats available and price is fine → confirm
    print("Booking confirmed")
else:                                       # seats exist but price needs approval
    print("Confirm with customer")
Terminal $ python book_seat.py
Cannot book

Using or in the first condition means either problem — no seats or too expensive — will block the booking. This is the same pattern booking systems use to prevent reservations that can't actually be fulfilled.

Common Mistakes

Frequent errors and how to avoid them

Forgetting the Colon

Every if, elif, and else line must end with a colon. Omitting it causes a SyntaxError.

SyntaxError

Incorrect 

if ticket_type == "Premium"
    print("VIP access")

Correct 

if ticket_type == "Premium":
    print("VIP access")

Python uses the colon to signal that an indented block is about to begin. Think of it as the "open door" for the if statement's code.

Missing or Inconsistent Indentation

Python uses indentation to know which lines belong to the if block. Any deviation causes an IndentationError.

IndentationError

Incorrect 

if score >= 50:
print("You passed")

Correct 

if score >= 50:
    print("You passed")

Use exactly four spaces per indentation level. Most code editors insert four spaces automatically when you press the Tab key inside a Python file.

Using = Instead of == in a Condition

A single = assigns a value. A double == compares two values. These are completely different operations.

SyntaxError

Incorrect 

if ticket_type = "Premium":   # SyntaxError
    print("VIP access")

Correct 

if ticket_type == "Premium":  # comparison
    print("VIP access")

Inside an if condition, always use == to compare. Save the single = for assignment statements such as ticket_type = "Premium".

Real-World Use

How if statements power everyday decisions in Python

Every app you use is making decisions in real time — and those decisions are written as if statements. A streaming service checks your subscription tier before it plays a show. A cinema sets the ticket price based on the customer's age. A courier app routes a parcel to the right delivery band based on weight. You write if condition: once, and Python follows the right path automatically.

One account,
three subscription tiers

When a user presses play, Python checks plan_type against each tier — "Basic", "Standard", or "Premium" — and unlocks the right content quality. One if / elif / else block handles all three paths.

returns content_quality

Six seats sold,
three different prices

A cinema checkout checks whether age < 12 (child), age >= 65 (senior), or neither (adult) and sets the price accordingly. Exactly one branch runs — the first condition that evaluates to True.

returns ticket_price

A parcel packed,
one courier to choose

A logistics app checks weight <= 1 for standard post, weight <= 5 for courier, and falls back to else for freight. Python reads each condition in order and stops at the first match.

returns "standard" / "courier" / "freight"

Without if statements

You manually look up each user's account tier and decide which content they can access — one lookup per user, easy to get wrong at scale.

A cashier reads the customer's age, picks the price from a printed chart, and keys it in — the same manual decision for every single ticket sold.

A warehouse worker weighs each parcel, consults a rate card, and selects a courier by hand — slow, inconsistent, and error-prone at scale.

With if statements

if plan_type == "Premium": — Python checks once and returns the correct content quality instantly, for every user, every time.

if age < 12: — Python branches to child, senior, or adult pricing in microseconds, with no chart and no human error.

if weight <= 1: — Python reads the weight, picks the first matching bracket, and assigns a courier automatically — no rate card, no guesswork.
