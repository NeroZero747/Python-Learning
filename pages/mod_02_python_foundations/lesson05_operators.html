Lesson Objective

The goal and expected outcome of this lesson

What operators are

You can recognize the symbols Python uses to perform actions on values — like +, ==, and and.

Arithmetic operators

You can use +, -, *, and / to calculate totals, differences, and averages in your script.

Comparison operators

You can use ==, >, and < to test a value and get back True or False.

Logical operators

You can use and, or, and not to combine conditions and make smarter decisions in your code.

This lesson covers the three operator types you'll use in almost every Python script — arithmetic, comparison, and logical.

Overview

A high-level summary of the topic

Py

Operators are symbols that tell Python what action to perform on one or more values.

Think of operators like the keys on a cashier's till: each key does one specific job — add an item, apply a discount, check whether a voucher is valid, or save the running total. A Python operator is that key, and your script is the till program that uses them to get a result.

Arithmetic operators

The math keys — add, subtract, multiply, divide

The till keys that crunch the numbers on every transaction.

Comparison operators

The price-check key — is this more, less, or equal?

The till button that checks whether a price meets a condition.

Logical operators

The multi-rule key — valid only if both checks pass

The till logic that combines two conditions into one decision.

Assignment operators

The "save total" key — stores the result for later

The till key that records the running total into memory.

If you've written a formula in Excel — such as =A1+B1 or =IF(A1>100,"High","Low") — you've already used the same three operator types; Python just writes them as code instead of cell references.

Key Concepts

Arithmetic, comparison, logical, and assignment operators — with examples for each.

Arithmetic Operators

Mathematical calculations on numbers

math ops

Arithmetic operators are symbols that perform calculations on numbers, just like the keys on a calculator. Python supports addition, subtraction, multiplication, division, floor division, and the modulo remainder.

Arithmetic Operators

Op Meaning Example
+ Add 3 + 4 → 7
- Subtract 10 - 6 → 4
* Multiply 5 * 3 → 15
/ Divide 10 / 4 → 2.5
// Floor divide 7 // 2 → 3
% Remainder 7 % 2 → 1
Python 
price = 49.99        # a decimal number (the unit price)
quantity = 3         # a whole number (items in the order)

total = price * quantity   # multiply to get the gross total = 149.97
discount = total * 0.1     # calculate 10% discount = 14.997
final = total - discount   # subtract discount from total = 134.973
print(round(final, 2))     # prints 134.97 (rounded to 2 decimal places)

Use // for whole-number division: 7 // 2 gives 3, not 3.5 — it drops the decimal, which is useful when counting items that cannot be split, such as seats or tickets.

Comparison Operators

Test relationships between values

bool result

Comparison operators test the relationship between two values and always return a Boolean result — either True or False. They are the building blocks of every decision in Python.

Comparison Operators

Op Meaning Example
== Equal to 5 == 5 → True
!= Not equal to 5 != 3 → True
> Greater than 10 > 5 → True
< Less than 3 < 8 → True
>= Greater or equal 5 >= 5 → True
<= Less or equal 4 <= 7 → True
Python 
sales = 120              # today's sales figure

print(sales > 100)       # True — sales exceeded the target
print(sales == 120)      # True — exact match confirmed
print(sales >= 100)      # True — met or exceeded the target
print(sales != 50)       # True — sales are definitely not 50
print(sales < 200)       # True — still under the ceiling limit

== compares, = assigns: writing sales = 100 stores 100 in the variable, but sales == 100 tests whether it equals 100 — confusing the two is one of the most common beginner mistakes in Python.

Logical Operators

Combine conditions with and / or / not

bool ops

Logical operators — and, or, and not — combine or reverse Boolean conditions so Python can evaluate two or more tests at once. They are essential for writing if statements that check multiple criteria.

Logical Operators

Op Meaning Example
and Both must be True True and True → True
or Either can be True False or True → True
not Flip True ↔ False not False → True
Python 
age = 28                              # user's age
has_account = True                    # whether the user is registered

is_eligible = age >= 18 and has_account   # both conditions must be True
is_guest    = not has_account             # flip: True becomes False
can_access  = age >= 18 or has_account    # either condition is enough

print(is_eligible)                        # True
print(is_guest)                           # False

and is stricter than or: A and B returns False the moment either side is False, while A or B only returns False when both sides are False.

Assignment Operators

Store and update values in memory

assign ops

Assignment operators store a value in a variable. The basic operator = saves a value for the first time, while shorthand operators like += and -= update an existing value without rewriting the full expression.

Assignment Operators

Op Meaning Example
= Assign a value score = 0
+= Add and update score += 10 → 10
-= Subtract and update score -= 3 → 7
*= Multiply and update score *= 2 → 14
/= Divide and update score /= 2 → 7.0
Python 
score = 0            # basic assignment: start the score at zero

score += 10          # same as: score = score + 10  →  10
score += 5           # same as: score = score + 5   →  15
score -= 3           # same as: score = score - 3   →  12
score *= 2           # same as: score = score * 2   →  24

print(score)         # prints 24

+= is the go-to shorthand for running totals: instead of writing total = total + sale_amount every time, write total += sale_amount — it's shorter, cleaner, and exactly what you will use in loops to accumulate values.

Code Examples

Operators in everyday Python examples

01

Calculate a Price

Beginner Shopping *, -, / Arithmetic

What This Does

This script uses arithmetic operators to work out a sale price — multiplying to get a subtotal, subtracting a discount, then dividing to find the new price per item. Change any number at the top and Python recalculates everything below automatically.

calculate_price.py 
item_name = "headphones"           # the item you are buying
item_price = 80                    # original price in dollars
quantity = 2                       # how many you are buying
discount = 10                      # discount amount in dollars

subtotal = item_price * quantity   # multiply price by quantity = 160
total = subtotal - discount        # subtract the discount = 150
price_per_item = total / quantity  # divide to find the new per-item price = 75.0

print("Item:", item_name)          # display the item name
print("Subtotal: $", subtotal)     # display 160
print("Total: $", total)           # display 150
print("Per item: $", price_per_item) # display 75.0
Terminal $ python calculate_price.py
Item: headphones
Subtotal: $ 160
Total: $ 150
Per item: $ 75.0

Change item_price or quantity at the top and every result below updates automatically — this is what makes variables so powerful.

02

Compare a Score

Beginner Quiz >=, ==, > True / False

What This Does

This script uses comparison operators to test a quiz score against three thresholds. Each comparison produces True or False — the result Python uses whenever it needs to make a decision.

compare_score.py 
my_score = 72          # the score you got on the quiz
pass_mark = 60         # the minimum score needed to pass
top_score = 100        # the highest possible score

passed = my_score >= pass_mark         # True if score is at or above the pass mark
got_full_marks = my_score == top_score # True only if score is exactly 100
above_halfway = my_score > 50          # True if score is more than half marks

print("My score:", my_score)           # display your score
print("Passed:", passed)               # True — 72 is at or above 60
print("Full marks:", got_full_marks)   # False — 72 is not 100
print("Above halfway:", above_halfway) # True — 72 is more than 50
Terminal $ python compare_score.py
My score: 72
Passed: True
Full marks: False
Above halfway: True

Every comparison always produces True or False — these are the results that power every if statement you will write in the lessons ahead.

03

Check Two Conditions

Beginner Plans and, or, not Logical

What This Does

This script uses logical operators — and, or, and not — to combine comparison results. Use them whenever a decision depends on more than one condition at once.

check_conditions.py 
have_ticket = True       # whether you already have a cinema ticket
film_is_on = True        # whether the film is showing tonight
is_raining = False       # whether it is raining outside

can_go = have_ticket and film_is_on     # True only if BOTH conditions are True
stay_in = is_raining or not have_ticket # True if raining, or if you have no ticket
good_weather = not is_raining           # flips False to True — it is NOT raining

print("Have ticket:", have_ticket)      # display whether you have a ticket
print("Can go to cinema:", can_go)      # True — both are True
print("Stay in tonight:", stay_in)      # False — not raining and you have a ticket
print("Good weather:", good_weather)    # True — not is_raining flips False to True
Terminal $ python check_conditions.py
Have ticket: True
Can go to cinema: True
Stay in tonight: False
Good weather: True

and only gives you True when every condition is True — if even one is False, the result is False.

Most common beginner confusion

In SQL and Excel, a single = means equals. In Python, a single = means assignment (store a value). To test equality in Python you must write == with two equals signs.

In Excel, AND() and OR() are worksheet functions — they take arguments in parentheses. In Python and SQL, and / AND are keywords placed between conditions, with no parentheses required.

Practice Exercises

Guided exercises to reinforce your learning

01

Calculate an Order Total

Beginner Shopping *, -, + Arithmetic

Your Task

You are buying 2 pairs of headphones at £45 each. Because your order is over £50, a 10% discount applies. There is also a flat delivery fee of £3.99. Write a Python program that stores the price, quantity, discount rate, and delivery fee as variables, calculates the subtotal, the discount amount, and the final order total, and prints all three values.

order_total.py 
price = 45.00            # price of a single pair of headphones
quantity = 2             # number of items in the order
discount_rate = 0.10    # 10% discount because order exceeds £50
delivery_fee = 3.99     # flat delivery charge

subtotal = price * quantity                             # multiply price by qty = 90.00
discount_amount = subtotal * discount_rate              # 10% of subtotal = 9.00
order_total = subtotal - discount_amount + delivery_fee # subtract discount, add delivery = 84.99

print("Subtotal: £", subtotal)           # display the pre-discount total
print("Discount: £", discount_amount)    # display the amount saved
print("Order total: £", order_total)     # display the final amount to pay
Terminal $ python order_total.py
Subtotal: £ 90.0
Discount: £ 9.0
Order total: £ 84.99

Storing the discount rate and delivery fee as named variables means that if those values ever change, you only update them in one place — the calculation on the next line adjusts automatically.

02

Check a Quiz Score

Beginner Quiz >=, == Comparison

Your Task

A student answered 18 out of 25 quiz questions correctly. The pass mark is 60% and a distinction is awarded for 80% or above. Write a Python program that calculates the student's percentage score, then uses comparison operators to check whether the student passed, whether they achieved a distinction, and whether they scored a perfect 100%. Print the percentage score and all three results.

quiz_score.py 
correct = 18             # number of questions answered correctly
total = 25               # total number of questions in the quiz
pass_mark = 60           # minimum percentage needed to pass
distinction_mark = 80   # minimum percentage needed for a distinction

score = (correct / total) * 100              # calculate percentage score = 72.0

passed = score >= pass_mark                  # True if score is 60% or above
got_distinction = score >= distinction_mark  # True if score is 80% or above
is_perfect = score == 100                    # True only if every question was correct

print("Score:", score, "%")                  # 72.0 %
print("Passed:", passed)                     # True
print("Distinction:", got_distinction)       # False
print("Perfect:", is_perfect)                # False
Terminal $ python quiz_score.py
Score: 72.0 %
Passed: True
Distinction: False
Perfect: False

In a real application, the True and False values that comparison operators return are what drive automated decisions — like showing a "Congratulations, you passed!" message or routing the student to a remedial module — without a person checking each score individually.

03

Plan a Trip Budget

Beginner Travel +=, -= and, or Logical

Your Task

You start with a travel budget of £500. You spend £180 on flights, then £140 on a hotel. A friend joins and splits the hotel cost with you (you get £70 back). You also receive a £20 voucher. Use augmented assignment operators to update your budget at each step. Then use logical operators to check whether you can afford a £90 activity (budget covers it and is still positive) and whether you should pack extra cash (budget is under £100 or the activity costs over £80). Print your remaining budget and both results.

trip_budget.py 
budget = 500.00          # starting travel budget
activity_cost = 90.00   # cost of the planned activity

budget -= 180.00         # pay for flights -- budget becomes 320.00
budget -= 140.00         # pay for hotel -- budget becomes 180.00
budget += 70.00          # friend splits hotel cost -- budget becomes 250.00
budget += 20.00          # voucher credited -- budget becomes 270.00

can_afford_activity = budget >= activity_cost and budget > 0  # both must be True
pack_extra_cash = budget < 100 or activity_cost > 80           # either condition triggers this

print("Remaining budget: £", budget)               # £270.0
print("Can afford activity:", can_afford_activity)  # True
print("Pack extra cash:", pack_extra_cash)           # True (activity_cost 90 > 80)
Terminal $ python trip_budget.py
Remaining budget: £ 270.0
Can afford activity: True
Pack extra cash: True

Using += and -= updates a single budget variable step by step, so you can read the code from top to bottom and clearly see every deduction and credit that was applied — exactly like reading a bank statement.

Common Mistakes

Pitfalls beginners hit when working with operators

Using a Single = to Test Whether Two Values Are Equal

A single = stores a value; == asks whether two values are equal — using the wrong one produces a silent wrong answer instead of True or False.

Pitfall

In Python, = is the assignment operator — it stores a value into a variable, not ask a question. Writing is_match = score = target chains two assignments, so is_match ends up holding the numeric value of target, not True or False. Replace the inner = with == to get a proper comparison result.

Wrong — assignment instead of comparison 

score = 85
target = 85
is_match = score = target  # assigns target to score, not a comparison
print(is_match)            # prints 85, not True or False

Correct — double == for comparison 

score = 85
target = 85
is_match = score == target  # compares the two values
print(is_match)             # prints True

A useful memory aid: = is like writing a value onto a sticky note — you are recording a fact. == is like looking at two sticky notes and asking "do they say the same thing?" — you are asking a yes/no question.

Assuming Python Reads Arithmetic Left to Right

Python follows BODMAS — multiplication and division always run before addition and subtraction, regardless of where they appear in the expression.

Pitfall

When Python evaluates price + delivery * 3, it multiplies delivery * 3 first, then adds price — even though addition appears earlier in the expression. To add first, wrap the addition in parentheses: (price + delivery) * 3.

Wrong — precedence ignored 

price = 12
delivery = 3

total = price + delivery * 3   # intends (12+3)*3 = 45
print("Total: $", total)        # prints 21, not 45

Correct — parentheses control order 

price = 12
delivery = 3

total = (price + delivery) * 3  # add first, then multiply
print("Total: $", total)         # prints 45

Parentheses in Python work exactly like they do in an Excel formula — anything inside them is calculated first. When an arithmetic expression gives you an unexpected number, adding parentheses around the addition step is the first fix to try.

Leaving Out the Variable Name in a Logical Chain

Python cannot infer which variable the second condition applies to — every condition joined by and or or must name its variable explicitly.

Pitfall

Writing score > 50 and < 90 reads naturally in English but raises a SyntaxError because the second half, < 90, has no left-hand operand — Python cannot infer that you still mean score. Both sides of and must be complete, self-contained comparison expressions.

Wrong — incomplete second condition 

score = 72

in_range = score > 50 and < 90  # SyntaxError — < 90 has no left operand

Correct — full variable on each side 

score = 72

in_range = score > 50 and score < 90  # each side is a complete comparison
print("In range:", in_range)           # prints True

Every condition in a logical chain must stand on its own as a valid Python expression — think of each side of and as a separate question Python asks independently: "Is score greater than 50?" and "Is score less than 90?" — both need the variable stated explicitly.

Real-World Use

How operators are used across everyday tasks

Every time you use an app, an operator is working behind the scenes. Shopping sites calculate totals, quiz apps grade your answers, and travel tools check your budget — all with arithmetic, comparison, and logical operators. Python gives you the same tools to build those rules yourself.

returns True or False

A traveller tracking
a £500 budget

Without operators

You open a calculator, work out each line item by hand, and type the total into a cell — one mistake means the whole order is wrong.

You read every answer, count the correct ones, divide on a calculator, and manually decide pass or fail — 50 answers means 50 steps.

You subtract each expense on paper and check two separate questions — it's easy to lose track of the running total or miss a condition.

With operators
