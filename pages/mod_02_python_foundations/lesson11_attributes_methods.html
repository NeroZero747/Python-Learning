Lesson Objective

The goal and expected outcome of this lesson

What attributes are

Attributes are variables that belong to an object, storing its specific data.

What methods are

Methods are functions defined inside a class that let an object perform actions.

Attributes give objects memory

Each object you create holds its own separate copy of every attribute you define.

Methods act on that data

A method reads or updates an object's attributes every time you call it.

This lesson gives you the building blocks to model real healthcare records — storing a member's details as attributes and writing methods that check eligibility or calculate costs.

Overview

A high-level summary of the topic

Py

Every object in Python has two things: data it remembers and actions it can perform.

Think of a vending machine: each one stores its own data — the item name, the price, how many are left — and it knows how to do things, like dispense an item or calculate your change. In Python, stored data is called an attribute and a built-in action is called a method.

Attribute

The display screen — shows what this machine stores

The screen shows only this machine's own name, price, and stock count.

Method

The button panel — each button triggers one action

Each button makes the machine carry out exactly one specific job.

Object Memory

Each machine stands alone — its own stock, its own data

Two machines of the same model can hold completely different item counts.

Methods Use Data

Press buy — the machine checks its own screen first

Every action the machine takes pulls from its own stored figures.

If you already use Excel, think of attributes as the values in a row and methods as the formulas that calculate with them — Python bundles both together inside one object.

Key Concepts

Attributes, methods, self, and __init__ — the four building blocks of every class.

Attribute

A named value stored inside every object

self.x

An attribute is a named value stored inside an object. You create one with self.name = value inside the class, and Python keeps a separate copy for every object you create.

Attribute Rules

Always prefix with self. self.name ✓ name ✗
Lowercase + underscores item_price ✓ ItemPrice avoid
Define in __init__ self.price = 0 inside the constructor ✓
Python 
class Product:
    def __init__(self):
        self.name     = "Laptop"   # stores a text value on this object
        self.price    = 999.99     # each object gets its own separate copy
        self.in_stock = True       # attributes can hold any data type

Each object keeps its own copy: changing laptop.price never affects phone.price — they are completely independent.

Method

A function that belongs to the class

def method()

A method is a function defined inside a class. It always takes self as its first parameter so it can read and update the object's own attributes.

Method Rules

Start with the def keyword def describe(self): ✓
First parameter is self def describe(self): ✓ def describe(): ✗
Indent the body 8 spaces inside the class return "value" inside the method ✓
Python 
class Product:
    def __init__(self):
        self.name     = "Laptop"       # attribute — the product name
        self.in_stock = True           # attribute — tracks availability

    def describe(self):                # define a method — self is first
        if self.in_stock:              # read own attribute via self
            return self.name + " is available"   # return a result
        return self.name + " is out of stock"    # alternative result

Call a method on the object: write laptop.describe() — Python passes the object as self automatically, so you never type self when calling.

self

The object's own reference to itself

self

self is the object that called the method. Python passes it automatically — you list it as the first parameter and use it to reach any attribute or method on the same object.

self vs the variable name

Inside the class Your script
Access attribute self.name laptop.name
Call method self.describe() laptop.describe()
Meaning "this object" "the laptop variable"
Python 
class Product:
    def __init__(self, name):
        self.name = name           # self stores the value on this object

    def show_name(self):
        print(self.name)           # self reads the stored value back

laptop = Product("Laptop")         # Python sets self = laptop automatically
laptop.show_name()                 # prints: Laptop

Never forget self in the parameter list: writing def show_name(): instead of def show_name(self): causes a TypeError when you call the method — Python cannot pass the object without it.

__init__

Runs automatically when an object is created

constructor

__init__ is the constructor method that runs the moment you create an object. Use it to set every attribute your object needs — think of it as filling in a form the very first time you create a record.

__init__ Rules

Name must be exactly __init__ __init__ ✓ init ✗ _init_ ✗
First parameter is self def __init__(self): ✓
Set all attributes here, not later self.price = 0 in __init__ ✓
Python 
class Product:
    def __init__(self, name, price):   # constructor — runs on creation
        self.name     = name           # set attribute from the argument
        self.price    = price          # set second attribute from argument
        self.in_stock = True           # default value — no argument needed

laptop = Product("Laptop", 999.99)     # __init__ runs automatically
print(laptop.price)                    # 999.99

Think of __init__ like filling in a form: every parameter you pass becomes an attribute — like typing a value into each field when you create a new record.

Decision Flow

A visual walkthrough of the logic and process

When you write your first Python class, three things happen in a strict, fixed order: you define the blueprint, you create objects from it, and then you call methods on those objects. This sequence — define → create → call — never changes, and Python reads it top to bottom, like a technician following a step-by-step protocol.

Inside a Python Class — Two Roles, One Blueprint

Attributes

Store data on the object

An attribute is a variable that belongs to exactly one object. You create attributes inside __init__ by writing self.x = value — after that, the object remembers that value forever and every method on the class can read it.

Attribute definition 
def __init__(self, name, price):
    self.name     = name    # ← attribute
    self.price    = price   # ← attribute
    self.in_stock = True    # ← attribute (default)
self.x = value lives in __init__ one copy per object

Methods

Define actions on the object

A method is a function written inside the class body. It always receives self as its first parameter, which is Python's way of handing the method a reference to the specific object it is working on. Use self.x inside the method to read or update any attribute.

Method definition 
def describe(self):       # ← method definition
    return (              #   self gives access to
        self.name +       #   ← reads attribute
        " — $" +
        str(self.price)   # ← reads attribute
    )
def name(self): reads attributes self is automatic

The bridge is self — when you call a method, Python automatically sets self to the object you called it on, giving the method instant access to every attribute that object has stored.

Class-First Order: Python reads your entire class definition when the file runs — before any object is created. That means the class block must appear above any line that calls Product(...). If you define the class after you try to use it, Python raises a NameError.

Read the comment on every line — each one labels exactly which step in the diagram above is happening at that moment:

product.py 
class Product:                               # Step 1 — define the blueprint
    def __init__(self, name, price):         # Step 2 — set starting values for every new object
        self.name     = name                 # attach name to this specific object
        self.price    = price                # attach price to this specific object
        self.in_stock = True                 # every new product starts as available

    def describe(self):                      # Step 3 — write a method (defined, not called yet)
        return self.name + " — $" + str(self.price)  # reads attributes on this object

item = Product("Laptop", 999.99)             # Step 4 — create one object; __init__ runs now
print(item.describe())                       # Step 5 — call the method; prints the label
Terminal $ python product.py
Laptop — $999.99

The Three Building Blocks of Every Class

Every Python class uses these three ideas together — none of them works correctly without the other two.

Attributes

The object's memory

An attribute is a variable that belongs to one specific object. You create every attribute by writing self.name = value inside __init__, and each object remembers its own separate copy forever.

Methods

The built-in actions

A method is a function defined inside a class. It always receives self as its first parameter so it can read and update the object's stored attributes whenever it runs.

Objects

One filled-in copy of the blueprint

An object is one live instance of your class, created by calling the class name like a function: item = Product(...). Each object holds its own independent data and can call any method the class defines.

Every class you write follows three layers: the blueprint (class), the data setup (__init__), and the actions (methods). Build these three layers in order and your code will always be easy to read, test, and reuse.

Store a Product

What This Does

This script defines a class with three attributes and creates one object from it. Every attribute is set inside __init__ the moment the object is created, and reading any attribute back is a single dot-access.

02

Update a Score

Beginner Students Methods Updating Attributes

What This Does

This script defines a class with two methods — one that changes an attribute and one that reads it. Calling the second method before and after the first shows how a method can update an object's stored data in place.

update_score.py 
class Student:                              # one blueprint for any student
    def __init__(self, name, score):        # set the student's data at creation time
        self.name  = name                   # store the student's name
        self.score = score                  # store their starting score

    def add_points(self, points):           # method to increase the stored score
        self.score = self.score + points    # update the score attribute on this object

    def grade(self):                        # method to read the score and return a label
        if self.score >= 70:               # check the stored score on this object
            return "Pass"                   # return Pass if the score is high enough
        return "Fail"                       # return Fail if the score is too low

student = Student("Alex", 55)              # create one object; score starts at 55
print(student.grade())                     # read the grade before adding points
student.add_points(20)                     # call add_points -- score updates to 75
print(student.grade())                     # read the grade again -- score has changed
Terminal $ python update_score.py
Fail
Pass

A method that changes an attribute and a method that reads it work as a natural pair — this "change then read" pattern appears in almost every real Python script that manages data.

03

List All Books

Beginner Books One Class, Many Objects List + Loop

What This Does

This script creates three separate objects from the same class and stores them in a list. A loop then calls the same method on every object — showing how one blueprint can produce as many independent records as you need.

list_books.py 
class Book:                                         # one blueprint for any book
    def __init__(self, title, author):              # set the book's data at creation time
        self.title  = title                         # store the book title
        self.author = author                        # store the author's name

    def summary(self):                              # method to print one line about this book
        print(self.title + " by " + self.author)   # reads this object's own attributes

books = [                                           # a list of three separate objects
    Book("The Pragmatic Programmer", "Andrew Hunt"),  # first object -- its own data
    Book("Clean Code", "Robert Martin"),              # second object -- independent
    Book("Python Crash Course", "Eric Matthes"),      # third object -- same blueprint, new data
]

for book in books:                                  # loop through every object in the list
    book.summary()                                  # call summary() on this specific object
Terminal $ python list_books.py
The Pragmatic Programmer by Andrew Hunt
Clean Code by Robert Martin
Python Crash Course by Eric Matthes

You only write the class once — add more items to the list and the loop handles every new object automatically, with no extra code needed.

Common Mistakes

Pitfalls beginners hit when working with attributes and methods

Forgetting self as the First Parameter of __init__

Without self, Python raises a TypeError the moment you try to create an object.

Pitfall

Every method in a class — including __init__ — must have self as its first parameter; without it, Python raises TypeError: __init__() takes 0 positional arguments but 1 was given when you call the class. Add self as the first parameter of every method you write.

Wrong — self omitted 

class Product:
    def __init__(name, price):
        self.name = name
        self.price = price

Correct — self first 

class Product:
    def __init__(self, name, price):  # self is always first
        self.name = name              # attach name to this object
        self.price = price            # attach price to this object

Think of self as the blank label Python attaches to each new object — without it, Python has no way to know which specific Product it's working with when you call Product("Laptop", 999).

Reading a Plain Variable Name Instead of the Object's Attribute

Writing name inside a method instead of self.name raises a NameError because that variable doesn't exist there.

Pitfall

Inside a method, Python only knows about self and any parameters you pass in — writing return name raises NameError: name 'name' is not defined because name was stored on the object, not left as a loose variable. Prefix every attribute with self. to read it.

Wrong — plain variable 

def describe(self):
    return name + " — $" + str(price)

Correct — use self. 

def describe(self):
    return self.name + " — $" + str(self.price)  # read attributes via self

Every method starts fresh with only self available — it's the key that unlocks all the values saved on that specific object, so always write self.name, never just name.

Defining a Method Outside the Class Body

A def at the top level becomes a plain function — it has no connection to the class and calling it on an object raises a TypeError.

Pitfall

Python uses indentation to decide what belongs inside a class — a def that starts at the leftmost column is a standalone function with no access to self or any class attributes. Indent every method four spaces inside the class body.

Wrong — method outside class 

class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

def describe(self):           # ← at top level, not inside class
    return self.name

Correct — indented inside class 

class Product:
    def __init__(self, name, price):
        self.name = name              # stored on this object
        self.price = price            # stored on this object

    def describe(self):               # 4-space indent — inside the class
        return self.name              # can now access self.name

Every line inside a class — including all its methods — must be indented exactly four spaces from the class keyword; if a def lines up with class, it belongs to the module, not the class.

Real-World Use

How classes, attributes, and methods are used across real-world workflows

Imagine you manage an online product catalog. Every product has a name, a price, and a stock status — and you need to display, update, and check each one. Without classes, every product is a loose collection of unconnected variables. With a class, you define the structure once and Python handles every product the same way — automatically, consistently, and with no repeated code.

Scenario — A Day Managing an Online Product Catalog

1

Morning — Stock the Catalog

Your supplier sends 200 new products. Using the Product class, you create one object per item — each one automatically stores a name, a price, and a starting stock status, all in one place.

laptop = Product("Laptop", 999.99)
phone  = Product("Phone",  699.00)
# Every object carries name, price, in_stock together
2

Midday — Print Product Labels

Your manager needs a formatted label for every product. You call describe() on each object — the method reads self.name and self.price automatically. No copy-pasting the format string.

print(laptop.describe())  # → Laptop — $999.99
print(phone.describe())   # → Phone — $699.0
# One method, reads every object’s own data
3

Evening — Run the Stock Report

Before closing, you run a stock report. The in_stock attribute lives on the same object as the name and price — all three values travel together and can never fall out of sync.

for p in catalog:
    if p.in_stock:
        print(p.name, "is available")
# attributes stay together — never out of sync

Why Classes Make This Work

One structure, zero repetition. Define Product once — every object carries the same fields automatically, with no copy-pasting.

Methods write themselves once. Change describe() in one place and every object in the catalog updates automatically — no tracking down repeated format strings.

Data stays together. name, price, and in_stock live on the same object and are always in sync — no separate lists or dictionaries to keep aligned.
