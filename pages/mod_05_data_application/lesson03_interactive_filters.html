This lesson teaches you to build interactive filters in Streamlit — from a single dropdown to a full sidebar-driven dashboard with charts that update on every user selection.

Overview

A high-level summary of the topic

Py

An interactive filter lets your dashboard users choose exactly which slice of data they want to see — no code changes, no new report requests.

Think of an interactive dashboard like a vending machine: the user presses buttons to select what they want, the machine delivers the matching item, and the display updates to show what was dispensed. Each Streamlit widget is one of those buttons.

Input Widgets

The button panel — every choice at your fingertips

Streamlit provides ready-made input components like st.selectbox(), st.slider(), and st.multiselect() that you drop into your script with a single function call. Each widget returns the value the user selected, which you store in a plain Python variable and use in the rest of your code.

DataFrame Filtering

The dispensing mechanism — delivers only what you asked for

Once you have the user's choice in a variable, you filter your pandas DataFrame with a boolean condition like df[df["region"] == selected_region]. The result is a new DataFrame containing only the rows that match, leaving the original data untouched.

Chained Filters

The combo order — press two buttons for a narrower pick

You can stack multiple widgets so each one narrows the data further — for example, first select a region, then slide a date range. Each filter applies to the already-filtered DataFrame, giving the user precise control over what they see.

Reactive Charts

The display screen — shows exactly what was dispensed

When you pass a filtered DataFrame to st.bar_chart() or st.line_chart(), Streamlit redraws the chart every time a widget value changes. The user sees an instant visual update without clicking a refresh button.

Sidebar Layout

The side panel — keeps buttons out of the product window

Placing filter widgets inside st.sidebar moves them to a collapsible left panel, keeping the main content area free for charts and tables. This separation makes dashboards easier to read, especially on smaller screens.

Interactive filters turn a static report into a self-service tool — stakeholders explore the data on their own instead of asking you to re-run queries.

Key Takeaways

The most important ideas to remember

Every Widget Returns a Python Value

When a user interacts with st.selectbox() or st.slider(), the widget immediately returns the selected value into a normal Python variable. You never need to write callback functions or event listeners — just assign the result and use it downstream.

st.selectbox()st.slider()return value

Filtering Creates a New DataFrame, Not a View

The expression df[df["col"] == value] returns a brand-new DataFrame, leaving the original intact. If you accidentally filter the original variable without re-assigning, subsequent widgets will operate on stale data and produce confusing results.

boolean masknew DataFrameassignment

Chained Filters Narrow Data Progressively

Each additional widget filters the already-filtered DataFrame, so the order you apply them matters. If the first filter removes all "West" rows, a second filter on city will never show cities from the West region.

progressive filteringorder mattersstacked widgets

Charts Redraw on Every Widget Change

Streamlit re-executes your entire script from top to bottom whenever a widget value changes. Any chart that references the filtered DataFrame automatically reflects the new selection without you writing update logic.

script re-runst.bar_chart()auto-refresh

Sidebar Keeps Filters Separate from Results

Moving widgets to st.sidebar prevents the filter controls from pushing your charts down the page. On mobile, the sidebar collapses into a hamburger menu, giving the user more screen space for data.

st.sidebarlayoutmobile-friendly

Key Concepts

Core terms and definitions for this topic

Input Widgets

Concept 1 of 5

Core

Streamlit input widgets are pre-built UI components that capture a user’s choice and return it as a Python value you can use immediately in your script.

Python
import streamlit as st
import pandas as pd

df = pd.read_csv("sales.csv")          # load your dataset

# Dropdown — returns the selected string
region = st.selectbox(
    "Select a region",                  # label shown to the user
    df["region"].unique()               # options pulled from the data
)

# Slider — returns the selected number
min_sales = st.slider(
    "Minimum sales amount",             # label
    min_value=0,                        # slider start
    max_value=int(df["sales"].max()),   # slider end
    value=0                             # default position
)
Common WidgetsDescription
st.selectbox()Single-choice dropdown; returns one value
st.multiselect()Multi-choice dropdown; returns a list
st.slider()Numeric or date slider; returns the selected value
st.date_input()Calendar date picker; returns a datetime.date
st.text_input()Free-text field; returns a string

Always pull dropdown options from the data itself (df["col"].unique()) instead of hard-coding a list. If the data changes, the widget stays in sync automatically.

Filtering DataFrames

Concept 2 of 5

Core

Filtering a DataFrame means using the value returned by a widget to create a boolean mask that selects only the rows you want to display or chart.

Python
# region comes from st.selectbox() above
filtered = df[df["region"] == region]   # boolean mask: True where region matches

st.write(f"Showing {len(filtered)} rows for {region}")
st.dataframe(filtered)                  # display the filtered table
Filter PatternsDescription
df[df["col"] == val]Exact match on a single value
df[df["col"].isin(vals)]Match any value in a list (multiselect)
df[df["col"] >= num]Numeric threshold (slider)
df[(cond1) & (cond2)]Combine two conditions with &

Store the filter result in a new variable (filtered = ...) instead of overwriting the original df. You will need the original data to populate widget options for subsequent filters.

Chaining Filters

Concept 3 of 5

Intermediate

Chaining filters means applying two or more widget selections in sequence so the displayed data narrows through multiple dimensions at once.

Python
# Widget 1 — region dropdown
region = st.selectbox("Region", df["region"].unique())

# Widget 2 — minimum sales slider
min_sales = st.slider("Min sales", 0, int(df["sales"].max()), 0)

# Apply both filters in order
filtered = df[df["region"] == region]          # first: region
filtered = filtered[filtered["sales"] >= min_sales]  # second: sales threshold

st.metric("Rows after filtering", len(filtered))
st.dataframe(filtered)
Chaining TipsDescription
Order mattersFilter from broad to narrow so each step has enough rows to be meaningful
Use & in one linedf[(df["region"]==r) & (df["sales"]>=n)] combines both conditions in a single expression
Show row countDisplay st.metric() or len(filtered) so users see how many rows survive each filter

If a filter combination returns zero rows, show a friendly st.warning("No data matches your filters.") instead of an empty chart — otherwise the user thinks the dashboard is broken.

Reactive Charts

Concept 4 of 5

Core

A reactive chart is one that automatically redraws itself whenever the user changes a filter widget, because Streamlit re-runs the entire script on every interaction.

Python
# filtered already contains only the matching rows
st.subheader(f"Sales in {region}")

# Bar chart — grouped by product
st.bar_chart(
    filtered.groupby("product")["sales"].sum()  # aggregate first
)

# Line chart — trend over time
st.line_chart(
    filtered.groupby("date")["sales"].sum()
)
Built-in Chart FunctionsDescription
st.bar_chart(data)Vertical bar chart; pass a Series or DataFrame
st.line_chart(data)Line chart; great for time-series trends
st.area_chart(data)Filled area chart; same API as line_chart
st.metric(label, value)Single KPI number with optional delta arrow

Group or aggregate the filtered data before charting (e.g., .groupby("product")["sales"].sum()). Passing raw row-level data to st.bar_chart() produces a bar for every row, which is usually unreadable.

Sidebar Layout

Concept 5 of 5

Layout

st.sidebar is a built-in Streamlit container that renders widgets in a collapsible left panel, keeping filter controls visually separate from the main results area.

Python
import streamlit as st
import pandas as pd

df = pd.read_csv("sales.csv")

# -- Sidebar filters --
st.sidebar.header("Filters")           # sidebar heading
region = st.sidebar.selectbox(          # widget inside sidebar
    "Region", df["region"].unique()
)
min_sales = st.sidebar.slider(
    "Min sales", 0, int(df["sales"].max()), 0
)

# -- Main area --
filtered = df[(df["region"] == region) & (df["sales"] >= min_sales)]
st.title("Sales Dashboard")
st.dataframe(filtered)
st.bar_chart(filtered.groupby("product")["sales"].sum())
Sidebar MethodsDescription
st.sidebar.selectbox()Dropdown inside the sidebar
st.sidebar.slider()Slider inside the sidebar
st.sidebar.header()Section heading in the sidebar
st.sidebar.markdown()Rich text inside the sidebar

On mobile devices the sidebar auto-collapses into a hamburger icon. Test your dashboard at a narrow viewport width to make sure labels and options still fit comfortably.

Code Examples

Hands-on code snippets to explore the concepts

dropdown_filter.py
import streamlit as st
import pandas as pd

df = pd.read_csv("sales.csv")               # load the dataset

region = st.selectbox(                       # create a dropdown widget
    "Select a region",                       # label the user sees
    df["region"].unique()                    # options from the data
)

filtered = df[df["region"] == region]        # filter rows by selection
st.dataframe(filtered)                       # show the filtered table

The dropdown automatically lists every unique value in the column. If the column has too many values (100+), consider st.text_input() with a search pattern instead.

slider_filter.py
import streamlit as st
import pandas as pd

df = pd.read_csv("sales.csv")

min_sales = st.slider(                       # numeric slider widget
    "Minimum sales amount",                  # label text
    min_value=0,                             # left end of slider
    max_value=int(df["sales"].max()),        # right end from data
    value=0                                  # default position
)

filtered = df[df["sales"] >= min_sales]      # keep rows above threshold
st.write(f"Rows with sales >= {min_sales}: {len(filtered)}")
st.dataframe(filtered)
Terminal$ streamlit run slider_filter.py
$ streamlit run slider_filter.py Local URL: http://localhost:8501 [Slider set to 500] Rows with sales >= 500: 189

Set a sensible default value for the slider so the dashboard loads with meaningful data rather than showing all rows or zero rows.

multiselect_filter.py
import streamlit as st
import pandas as pd

df = pd.read_csv("sales.csv")

products = st.multiselect(                   # multi-choice dropdown
    "Select products",                       # label
    options=df["product"].unique(),           # all choices
    default=df["product"].unique()[:3]       # pre-select first 3
)

if products:                                 # guard against empty list
    filtered = df[df["product"].isin(products)]  # .isin() for lists
    st.dataframe(filtered)
else:
    st.warning("Please select at least one product.")
Terminal$ streamlit run multiselect_filter.py
$ streamlit run multiselect_filter.py Local URL: http://localhost:8501 [Products: Widget, Gadget, Gizmo selected] Showing 456 rows

Always guard multiselect with an if-check. When the user clears all selections, .isin([]) returns zero rows and charts render empty — a warning message is much friendlier.

chained_filters.py
import streamlit as st
import pandas as pd

df = pd.read_csv("sales.csv")

# Two filters in the sidebar
st.sidebar.header("Filters")
region = st.sidebar.selectbox("Region", df["region"].unique())
min_sales = st.sidebar.slider("Min sales", 0, int(df["sales"].max()), 0)

# Apply both filters
filtered = df[(df["region"] == region) & (df["sales"] >= min_sales)]

# Main area — chart + table
st.title(f"Sales in {region}")
st.bar_chart(filtered.groupby("product")["sales"].sum())
st.dataframe(filtered)
Terminal$ streamlit run chained_filters.py
$ streamlit run chained_filters.py Local URL: http://localhost:8501 [Sidebar: Region=West, Min sales=200] Sales in West [Bar chart: Widget 1200, Gadget 850, Gizmo 640] Showing 127 rows

Placing both filters in the sidebar keeps the main area dedicated to results. The chart and table update together every time either filter changes.

full_dashboard.py
import streamlit as st
import pandas as pd

st.set_page_config(layout="wide")           # use full browser width
df = pd.read_csv("sales.csv")

# -- Sidebar filters --
st.sidebar.header("Dashboard Filters")
region = st.sidebar.selectbox("Region", ["All"] + list(df["region"].unique()))
products = st.sidebar.multiselect("Products", df["product"].unique(), default=df["product"].unique())
min_sales = st.sidebar.slider("Min sales", 0, int(df["sales"].max()), 0)

# -- Apply filters --
filtered = df.copy()                         # start with full data
if region != "All":
    filtered = filtered[filtered["region"] == region]
filtered = filtered[filtered["product"].isin(products)]
filtered = filtered[filtered["sales"] >= min_sales]

# -- KPI row --
col1, col2, col3 = st.columns(3)            # three metric columns
col1.metric("Total Sales", f"${filtered['sales'].sum():,.0f}")
col2.metric("Avg Sale", f"${filtered['sales'].mean():,.0f}")
col3.metric("Rows", len(filtered))

# -- Charts --
st.bar_chart(filtered.groupby("product")["sales"].sum())
st.dataframe(filtered)
Terminal$ streamlit run full_dashboard.py
$ streamlit run full_dashboard.py Local URL: http://localhost:8501 [Sidebar: Region=All, Products=all, Min sales=0] Total Sales: $124,500 | Avg Sale: $312 | Rows: 400 [Bar chart with all products]

Adding an "All" option to the dropdown with ["All"] + list(...) lets users see the unfiltered view without removing the dropdown entirely.

SQL / Excel Comparison

How this topic compares across Python, SQL, and Excel

Dropdown Filter
Pythonst.selectbox()

A dropdown that returns the selected value as a Python string you can use immediately.

SQLWHERE col = 'value'

SQL filters with a WHERE clause, but requires a new query each time.

ExcelData > Filter

Excel AutoFilter lets you pick values from a dropdown in the column header.

Boolean Mask
Pythondf[df["col"] == val]

Pandas boolean indexing selects rows where the condition is True.

SQLWHERE col >= 100

SQL uses comparison operators in the WHERE clause for the same effect.

ExcelCustom AutoFilter

Excel AutoFilter supports number comparisons like "greater than" via custom rules.

Chained Conditions
Pythondf[(c1) & (c2)]

Combine multiple conditions with & (and) or | (or) inside the brackets.

SQLWHERE c1 AND c2

SQL chains conditions with AND / OR keywords.

ExcelMultiple Filters

Apply AutoFilter on several columns; Excel combines them with AND logic.

Reactive Chart
Pythonst.bar_chart(filtered)

Chart redraws automatically on every widget change because Streamlit re-runs the script.

SQLBI Dashboard

Tools like Power BI or Tableau refresh visuals when a slicer changes.

ExcelPivotChart + Slicer

Excel PivotCharts linked to slicers update when you click a slicer button.

Sidebar Layout
Pythonst.sidebar.selectbox()

Moves widgets to a collapsible left panel, separating controls from content.

SQLParameters pane

SSRS and Power BI have a parameters panel separate from the report body.

Exceln/a

Excel does not have a sidebar concept; filters live inside the sheet.

Common Mistakes

Frequent errors and how to avoid them

Hard-coding dropdown options

Beginners type a manual list instead of pulling options from the data.

Pitfall

Wrong

st.selectbox("Region", ["West", "East"])

Correct

st.selectbox("Region", df["region"].unique())

If a new region appears in the data, the hard-coded list will miss it. Always use df["col"].unique() to keep the widget in sync with the actual data.

Overwriting the original DataFrame

Assigning the filter result back to df destroys the unfiltered data that other widgets need.

Pitfall

Wrong

df = df[df["region"] == region]  # overwrites df!

Correct

filtered = df[df["region"] == region]  # new variable

Always store filter results in a new variable. If you overwrite df, the next widget that calls df["col"].unique() will only see values from the already-filtered subset.

Empty multiselect crashes chart

When the user clears all multiselect options, .isin([]) returns zero rows and st.bar_chart() renders an empty, confusing visual.

Pitfall

Wrong

filtered = df[df["product"].isin(products)]
st.bar_chart(filtered)

Correct

if products:
    filtered = df[df["product"].isin(products)]
    st.bar_chart(filtered)
else:
    st.warning("Select at least one product.")

Always guard a multiselect filter with if products:. The one-line guard prevents empty charts and gives the user a clear next step.

Filtering before loading data

Placing the widget above pd.read_csv() causes a NameError because the DataFrame does not exist yet.

Pitfall

Wrong

region = st.selectbox("Region", df["region"].unique())
df = pd.read_csv("sales.csv")

Correct

df = pd.read_csv("sales.csv")
region = st.selectbox("Region", df["region"].unique())

Load data first, then create widgets that reference it. In Streamlit, code runs top to bottom on every interaction, so order matters.

Forgetting to aggregate before charting

Passing raw row-level data to st.bar_chart() creates one bar per row, producing an unreadable mess.

Pitfall

Wrong

st.bar_chart(filtered["sales"])  # one bar per row

Correct

st.bar_chart(filtered.groupby("product")["sales"].sum())

Group and aggregate (sum, mean, count) before charting. This gives you one bar per category, which is what users actually expect to see.

Real-World Use

Practical applications in real-world workflows

Sales Analytics

A regional manager selects their territory from a dropdown and sees only their team's revenue, targets, and top products — no SQL required.

Healthcare Dashboards

Hospital staff filter patient data by department and date range to track bed occupancy, average length of stay, and readmission rates.

Financial Reporting

Finance teams use a date slider and account-type multiselect to drill into monthly expenses, comparing actuals against budget in real time.

Marketing Campaigns

A marketing analyst filters by campaign name and channel to see click-through rates, conversion funnels, and spend versus return.

Supply Chain Monitoring

Operations managers filter shipments by warehouse and carrier to spot delivery delays and compare on-time performance.

Education Analytics

School administrators filter student data by grade level and subject to identify performance trends and allocate tutoring resources.
