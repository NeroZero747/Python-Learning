Core Concepts at a Glance

Three things to remember about variables and data types in Python.

Variables Store Information

Variables allow Python programs to store information so it can be reused later in the program.

Python 
age = 30
print(age)  # 30

The program can now use 30 whenever age is referenced.

Data Types Describe Values

Every value in Python has a data type. The type tells Python how the value should behave.

Python 
"Python"  # string
10        # integer
10.5      # float
True      # boolean

Numbers can be added. Text values can be combined.

Python Auto-Detects Types

Unlike some languages, Python does not require you to declare types manually — making it easier to learn and write.

Python 
name  = "Python"
count = 5
price = 19.99

Python automatically determines the type of each value.

Understanding these three concepts will help you read and write Python code with confidence from the very beginning.

Common Mistakes

Avoid these frequent errors that beginners make when working with variables and data types.

Mistake 1

Forgetting Quotes Around Strings

NameError

Incorrect 

name = Python

Correct 

name = "Python"

Without quotes, Python treats Python as a variable name. Since it doesn't exist, it raises a NameError. Always wrap text values in "quotes".

Mistake 2

Wrapping Numbers in Quotes

TypeError

Incorrect 

age = "30"

Correct 

age = 30

Quoting 30 stores it as the text "30", not a number. Any arithmetic like age + 5 will raise a TypeError at runtime.

Mistake 3

Incorrect Boolean Capitalization

NameError

Incorrect 

is_ready = true

Correct 

is_ready = True

Python is case-sensitive. true is an undefined variable, not a boolean — raising a NameError. Python's boolean literals must always start with a capital: True and False.
