SQL / Excel Comparison

How this topic compares across Python, SQL, and Excel

If you already store data in a SQL table or an Excel spreadsheet, you already know the idea behind attributes and methods — Python classes just let you keep that data and the actions that use it together in one place.

Python
SQL
Excel
store a value
Python attribute

An attribute stores one piece of data on an object, like product.price holding the number 999.99.

SQL column value

A column stores one value per row, like a price column in a products table.

Excel cell

A cell stores a single value, like the price you type into column B of your spreadsheet.

perform an action
Python method

A method is a function defined inside a class that reads the object's attributes and does something with them.

SQL stored procedure

A stored procedure is a saved query you call by name to carry out a repeatable task on your data.

Excel macro

A macro is a recorded sequence of steps Excel can replay with a single button click.

one complete record
Python object

An object holds all the attributes and methods for one specific thing, like one product in your store.

SQL row

A row in your products table holds all the column values for one product.

Excel row

A row in your spreadsheet holds all the fields for one item, like a product's name, price, and stock status.

Same product description, three tools

Python code

Python 
product = Product("Laptop", 999.99)
description = product.describe()
print(description)  # Laptop — $999.99

SQL query

SQL 
SELECT name || ' — $' || price
  AS description
FROM products
WHERE name = 'Laptop';

Excel formula

Excel 
=A2 & " — $" & B2
' A2 = Laptop, B2 = 999.99
' result: Laptop — $999.99

All three combine a product name and price into a readable description — the same result, just written in a different tool.

The data you store in SQL columns and Excel cells maps directly to attributes in Python, and the actions you run with stored procedures or macros map directly to methods — you are already thinking the right way, and Python just bundles them together inside a class.
