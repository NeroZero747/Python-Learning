01

Build a Product

Beginner Products __init__ Attributes

Your Task

Write a class called Product with an __init__ method that takes name and price as arguments and stores them as attributes. Create a Product object for a Coffee Maker priced at $49.99, then print both attributes on separate lines. Your output should show Coffee Maker on the first line and 49.99 on the second.

product_info.py 
class Product:                            # define the Product class
    def __init__(self, name, price):      # runs automatically when a Product is created
        self.name = name                  # store the name as an attribute
        self.price = price                # store the price as an attribute

coffee_maker = Product("Coffee Maker", 49.99)  # create a Product object
print(coffee_maker.name)                  # read and print the name attribute
print(coffee_maker.price)                 # read and print the price attribute
Terminal $ python product_info.py
Coffee Maker
49.99

Every object you create from Product gets its own copy of name and price — so a second product like a Keyboard would store its own values separately without affecting the Coffee Maker.

02

Add a Method

Beginner Students Methods if / else

Your Task

Write a class called Student with a name attribute and a score attribute. Add a method called result() that returns "Pass" if the score is 50 or above and "Fail" otherwise. Create a Student named Maria with a score of 72, then print her name and her result on separate lines.

student_result.py 
class Student:                            # define the Student class
    def __init__(self, name, score):      # __init__ takes name and score
        self.name = name                  # store the student's name
        self.score = score                # store their test score

    def result(self):                     # method that checks if the student passed
        if self.score >= 50:             # the pass mark is 50
            return "Pass"                 # score is high enough — return Pass
        else:
            return "Fail"                 # score is too low — return Fail

maria = Student("Maria", 72)             # create a Student with name and score
print(maria.name)                        # print the student's name
print(maria.result())                    # call result() and print the answer
Terminal $ python student_result.py
Maria
Pass

The result() method reads self.score so it always works with the specific student you call it on — if you create a second student with a different score, result() will check that student's score automatically, without any extra code from you.

03

Create Two Objects

Beginner Books Methods Multiple Objects

Your Task

Write a class called Book with a title attribute and an author attribute. Add a method called summary() that returns a string in the format "title by author". Create two Book objects — one for Clean Code by Robert Martin and one for Python Crash Course by Eric Matthes — then print both summaries.

book_summary.py 
class Book:                               # define the Book class
    def __init__(self, title, author):    # __init__ takes a title and an author
        self.title = title                # store the book title as an attribute
        self.author = author              # store the author name as an attribute

    def summary(self):                    # method that builds a readable description
        return self.title + " by " + self.author  # combine title and author into one string

book_1 = Book("Clean Code", "Robert Martin")         # create the first Book object
book_2 = Book("Python Crash Course", "Eric Matthes") # create the second Book object
print(book_1.summary())                  # call summary() on the first book
print(book_2.summary())                  # call summary() on the second book
Terminal $ python book_summary.py
Clean Code by Robert Martin
Python Crash Course by Eric Matthes

One class, two completely independent objects — book_1 and book_2 each store their own title and author, and calling summary() on either one uses that object's own data automatically.
